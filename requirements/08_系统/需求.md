# 系统模块需求文档

## 1. 模块定位

**一句话描述**：支撑App运行的基础能力，包括认证、同步、推送等。

**核心价值**：
- 安全：保护数据安全
- 可靠：确保数据不丢失
- 实时：及时同步数据

---

## 2. 认证系统

### 2.1 登录方式

| 方式 | 适用场景 | 优先级 |
|-----|---------|-------|
| 手机号+验证码 | 主要登录方式 | 高 |
| 手机号+密码 | 二次登录 | 中 |
| 微信一键登录 | 快捷登录 | 中 |
| 记住登录态 | 自动登录 | 高 |

### 2.2 登录流程

```
打开App
    ↓
检查本地登录态
    ↓
已登录 ──► 自动进入首页
    ↓
未登录/过期 ──► 进入登录页
    ↓
输入手机号
    ↓
获取验证码
    ↓
输入验证码
    ↓
验证成功
    ↓
获取用户信息
    ↓
进入首页
```

### 2.3 账号安全

**安全策略**：
- 验证码有效期：5分钟
- 验证码错误次数：最多5次
- 登录设备管理：可查看/踢出其他设备
- 异常登录提醒：新设备登录通知

## 3. 数据同步

### 3.1 同步策略

| 数据类型 | 同步时机 | 同步方向 |
|---------|---------|---------|
| 客户数据 | 实时 | 双向 |
| 订单数据 | 实时 | 双向 |
| 库存数据 | 实时 | 双向 |
| 安检数据 | 批量（定时） | 上传 |
| 财务数据 | 实时 | 双向 |
| 配置数据 | 变更时 | 双向 |

### 3.2 离线支持

**离线操作**：
- 离线可查看所有数据
- 离线可创建订单（本地队列）
- 离线可修改客户信息
- 离线可录入安检（待同步）

**同步机制**：
```
网络恢复
    ↓
检测待同步数据
    ↓
按优先级同步：
1. 订单状态变更（最高）
2. 新订单创建
3. 客户信息变更
4. 安检记录
    ↓
冲突处理：
- 库存：服务端优先
- 订单状态：本地优先
- 客户信息：时间戳优先
```

### 3.3 冲突解决

**冲突场景**：
```
场景1：离线时订单被修改
- 本地：修改订单数量
- 服务端：订单已被配送员完成
- 解决：服务端优先，提示用户

场景2：离线时库存变动
- 本地：创建订单锁定库存
- 服务端：库存已被其他订单占用
- 解决：服务端优先，提示库存不足

场景3：客户信息同时修改
- 本地：修改客户地址
- 服务端：修改客户电话
- 解决：合并修改，时间戳优先字段
```

## 4. 推送通知

### 4.1 推送类型

| 类型 | 触发条件 | 优先级 |
|-----|---------|-------|
| 库存预警 | 库存低于预警线 | 高 |
| 欠瓶提醒 | 欠瓶超期 | 高 |
| 欠款提醒 | 欠款超期 | 高 |
| 日结对账 | 每日21:00提醒 | 中 |
| 定期安检 | 客户待安检 | 中 |
| 系统通知 | 版本更新等 | 低 |

### 4.2 推送设置

```
推送总开关
    ↓
时段设置：08:00-20:00
    ↓
各类型开关：
- 库存预警 ☑
- 欠瓶提醒 ☑
- 欠款提醒 ☑
- 日结对账 ☑
- 定期安检 ☐
```

## 5. 数据备份

### 5.1 备份策略

| 备份类型 | 频率 | 保留时间 |
|---------|------|---------|
| 自动备份 | 每天23:00 | 30天 |
| 手动备份 | 用户触发 | 永久 |
| 换机备份 | 换机时 | 一次性 |

### 5.2 备份内容

- 客户数据
- 订单数据
- 库存数据
- 财务数据
- 配置数据
- 安检数据

---

## 6. 性能要求

| 指标 | 要求 |
|-----|------|
| 启动时间 | < 3秒 |
| 页面切换 | < 300ms |
| 数据加载 | < 500ms |
| 同步时间 | < 5秒（正常网络） |
| 离线支持 | 完整功能可用 |

---

## 7. 安全要求

- 数据传输：HTTPS加密
- 本地存储：敏感数据加密
- 访问控制：登录态验证
- 数据备份：云端加密存储
