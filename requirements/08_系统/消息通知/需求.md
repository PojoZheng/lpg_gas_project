# 消息通知需求文档

## 1. 功能定位

**一句话描述**：通过多渠道消息触达，及时提醒配送员待办事项和重要信息。

**核心价值**：
- 及时提醒：欠瓶、欠款、库存等关键事项不遗漏
- 多渠道触达：App内通知+推送+短信（重要时）
- 智能降噪：免打扰时段，避免过度打扰

---

## 2. 消息类型

### 2.1 业务提醒类

| 消息类型 | 触发条件 | 优先级 | 推送方式 |
|---------|---------|-------|---------|
| **欠瓶提醒** | 欠瓶超7/15/30天 | P0 | App内+推送 |
| **欠款提醒** | 欠款超7/15/30天 | P0 | App内+推送 |
| **库存预警** | 库存低于预警线 | P0 | App内+推送 |
| **日结对账** | 每日21:00 | P1 | App内+推送 |
| **定期安检** | 客户安检到期 | P1 | App内+推送 |

### 2.2 订单相关类

| 消息类型 | 触发条件 | 优先级 | 推送方式 |
|---------|---------|-------|---------|
| 订单取消 | 客户/系统取消订单 | P1 | App内 |
| 订单修改 | 订单信息被修改 | P2 | App内 |
| 新预约订单 | 客户预约未来配送 | P1 | App内+推送 |

### 2.3 系统通知类

| 消息类型 | 触发条件 | 优先级 | 推送方式 |
|---------|---------|-------|---------|
| 版本更新 | 有新版本发布 | P2 | App内+推送（可选） |
| 系统维护 | 计划维护通知 | P1 | App内+推送 |
| 账户安全 | 新设备登录等 | P0 | App内+推送+短信 |
| 政策变更 | 监管政策更新 | P1 | App内 |

## 3. 提醒机制

### 3.1 欠瓶/欠款提醒

```
┌─────────────────────────────────────────┐
│           欠瓶提醒规则                   │
├─────────────────────────────────────────┤
│                                         │
│  欠瓶产生                               │
│     │                                   │
│     ▼                                   │
│  T+0（当天）                             │
│     │ Toast "已记录欠瓶1瓶"              │
│     │                                   │
│     ▼                                   │
│  T+7（7天后）                            │
│     │ 推送："李大妈欠瓶已7天未还"        │
│     │ 首页：显示欠瓶待收徽章             │
│     │                                   │
│     ▼                                   │
│  T+15（15天后）                          │
│     │ 推送："欠瓶超期15天，建议转为收入" │
│     │ 首页：红色醒目提醒                 │
│     │                                   │
│     ▼                                   │
│  T+30（30天后）                          │
│     │ 按配置自动处理：                   │
│     │ • 转为收入 或 • 继续欠             │
│     │ 推送："欠瓶已自动转为收入"         │
│                                         │
└─────────────────────────────────────────┘
```

### 3.2 库存预警提醒

```
库存状态变化
    │
    ▼
正常（😊）→ 紧张（😐）
    │         │
    │    首页表情变化
    │    无需推送
    │
    ▼
紧张（😐）→ 不足（😰）
             │
             ▼
        首页表情变😰
        推送："15kg库存不足，仅剩2瓶"
        首页显示红色预警横幅
        开单时校验提示库存不足
```

### 3.3 日结对账提醒

```
每日21:00（可配置）
    │
    ▼
检查今日是否已对账
    │
    ▼
未对账 ──→ 推送："今日还有¥XXX待对账，记得核对"
    │           首页收入卡片显示红点
    │
已对账 ──→ 不推送
```

## 4. 推送设置

### 4.1 用户可控设置

```
┌─────────────────────────────────────────┐
│  推送设置                                │
├─────────────────────────────────────────┤
│                                         │
│  总开关                                 │
│  接收推送通知 [开关：开]                 │
│  （关闭后将不再接收任何推送）            │
│                                         │
│  ─────────────────────────────────────  │
│                                         │
│  免打扰时段                             │
│  开启免打扰 [开关：开]                   │
│  时段：[22:00] 至 [08:00]               │
│  （此时间段只接收P0紧急消息）            │
│                                         │
│  ─────────────────────────────────────  │
│                                         │
│  各类消息设置                            │
│                                         │
│  欠瓶/欠款提醒         [开关：开]        │
│  超期7/15/30天提醒                       │
│                                         │
│  库存预警              [开关：开]        │
│  库存不足时推送                          │
│                                         │
│  日结对账提醒          [开关：开]        │
│  每日21:00提醒（可改时间）               │
│                                         │
│  定期安检提醒          [开关：开]        │
│  客户安检到期前7天提醒                   │
│                                         │
│  订单通知              [开关：开]        │
│  新预约、订单取消等                      │
│                                         │
│  系统通知              [开关：开]        │
│  版本更新、政策变更等                    │
│                                         │
│  账户安全通知          [开关：开]        │
│  新设备登录等（建议开启）                │
│                                         │
└─────────────────────────────────────────┘
```

### 4.2 推送优先级与免打扰

| 优先级 | 消息类型 | 免打扰时段 |
|-------|---------|-----------|
| P0（紧急） | 欠瓶超30天、账户安全 | ✅ 仍推送 |
| P1（重要） | 库存不足、日结提醒 | ❌ 不推送，延后到08:00 |
| P2（普通） | 版本更新、系统通知 | ❌ 不推送，延后到08:00 |

## 6. 接口需求

### 6.1 获取消息列表

```http
GET /api/notifications?page=1&size=20&status=unread

Response:
{
  "code": 0,
  "data": {
    "total": 15,
    "unread_count": 3,
    "list": [
      {
        "id": "N001",
        "type": "owed_bottle",
        "title": "欠瓶提醒",
        "content": "李大妈欠瓶已超期7天",
        "data": {
          "customer_id": "C001",
          "customer_name": "李大妈",
          "owed_count": 1,
          "owed_days": 7
        },
        "priority": "high",
        "is_read": false,
        "created_at": "2026-04-07T10:30:00Z"
      }
    ]
  }
}
```

### 6.2 标记已读

```http
POST /api/notifications/read

Request:
{
  "ids": ["N001", "N002"]  // 不传则标记全部
}

Response:
{
  "code": 0,
  "message": "已标记为已读"
}
```

### 6.3 更新推送设置

```http
POST /api/settings/notifications

Request:
{
  "enabled": true,
  "quiet_hours": {
    "enabled": true,
    "start": "22:00",
    "end": "08:00"
  },
  "types": {
    "owed": true,
    "inventory": true,
    "settlement": true,
    "inspection": true,
    "order": true,
    "system": true
  }
}
```
