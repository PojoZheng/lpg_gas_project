## 2. 启动页（叙事版）

### 2.1 页面布局 - 故事卡片轮播

**设计理念**：用3张故事卡片代替传统的Logo+Slogan，让配送员一眼看到自己。

```

│  │     还有3家欠瓶要收...          │    │
│  │                                  │    │
│  │     以前带着笔记本和计算器      │    │

│                                         │
```

### 2.2 三张故事卡片内容

**卡片一：《晨光中的准备》**
```
场景：清晨装车
文案：
"清晨六点，你检查着车上的气瓶。
今天有18单要送，3家欠瓶要收，
还有那个总说'下次一起算'的王老板...

以前，你带着笔记本、计算器、
和一颗怕算错账的心。

现在，只需要这个App。"

画面：配送员装车的剪影 + 手机界面微光
```

**卡片二：《路上的从容》**
```
场景：配送途中
文案：
"电话响了，是李大妈：'小张啊，今天送气吗？'

你一边导航，一边看她的欠瓶记录：
'送！您欠的那瓶15kg的，今天也给您带过去。'

账在心里，清清楚楚。
不用翻本子，不用靠记忆。"

画面：导航界面 + 欠瓶提醒弹窗
```

**卡片三：《收工时的踏实》**
```
场景：晚上对账
文案：
"晚上八点，最后一单完成。
点开'今日收入'——¥1,680。

系统自动算好了收入、支出、利润。
终于不用再对着笔记本算半小时了。

每一单都算得清，每一天都赚得明。"

画面：收入统计页面 + 金币动画
```

### 2.3 启动流程（含叙事）

```
打开App
    ↓
显示第一张故事卡片（停留3秒，可滑动）
    ↓
用户滑动/自动切换 → 第二张 → 第三张
    ↓
点击【开始配送之旅】
    ↓
检查本地Token
    ↓
Token有效 ──→ 自动进入首页
    ↓
Token无效/过期 ──→ 显示登录页
```

### 2.4 交互细节

| 交互 | 效果 | 说明 |
|-----|------|------|
| 左右滑动 | 切换故事卡片 | 卡片间有视差滚动效果 |
| 底部圆点 | 指示当前卡片 | 当前卡片圆点高亮放大 |
| 自动播放 | 3秒自动切换 | 用户交互后暂停自动播放 |
| 跳过按钮 | 右上角【跳过】 | 点击直接进入登录/首页 |
| 开始按钮 | 底部大按钮 | 第三张卡片时按钮高亮跳动 |

### 2.2 启动流程

```
显示启动页（最少2秒，可跳过）
    ↓
并行执行：
├─ 检查版本更新
│   ├─ 有强制更新 → 跳转应用商店
│   └─ 有可选更新 → 提示，可跳过
│
├─ 检查本地Token
│   ├─ Token有效 ──→ 静默刷新Token ──→ 进入首页
│   └─ Token无效 ──→ 进入登录页
│
└─ 预加载首页数据（缓存）
```

### 3.2 登录页情感化设计

### 3.3 按钮状态规则

| 状态 | 手机号 | 验证码 | 登录按钮 | 获取验证码按钮 |
|-----|-------|-------|---------|--------------|

## 6. 动画效果

### 6.1 页面过渡动画

| 动画 | 触发 | 效果 | 时长 |
|-----|------|------|------|
| 启动页→登录页 | Token无效 | 启动页淡出，登录页淡入 | 300ms |
| 登录页→首页 | 登录成功 | 登录页向上滑出，首页从下方进入 | 400ms |
| 登录页→完善信息 | 新用户 | 右滑进入完善信息页 | 300ms |
| 完善信息→首页 | 完成 | 向上滑出，首页淡入 | 400ms |

### 6.2 元素动画

```
验证码按钮倒计时：
获取 → 点击 → 按钮变灰，显示"59s"
                ↓
         数字每秒递减（无动画）
                ↓
         倒计时结束 → 恢复"获取验证码"
                      带淡入效果（200ms）

登录按钮Loading：
点击登录 → 按钮文字变"登录中..."
            右侧显示旋转Loading图标
            按钮禁用（不可重复点击）
            ↓
         登录完成 → 恢复按钮状态
```

## 7. 安全规格

### 7.1 输入安全

| 安全项 | 处理方式 |
|-------|---------|
| 手机号显示 | 中间4位星号隐藏（138****5678） |
| 验证码输入 | 数字键盘，输入完成后自动校验长度 |
| 密码输入 | 可选显示/隐藏，默认隐藏 |
| 截图保护 | 登录页允许截图（无敏感信息） |

### 7.2 传输安全

- 所有接口HTTPS
- Token存储加密（Keychain/Keystore）
- 验证码明文传输（5分钟有效）
