# 登录认证需求文档

## 1. 功能定位与叙事设计

### 1.1 一句话描述

提供安全、便捷的用户身份认证能力，保障账户安全的同时，用温暖的开场白让配送员感受到被理解和被尊重。

**设计哲学**：不只是登录，是一次温暖的相遇。

### 1.2 启动页叙事文案

**【场景一：晨光中的准备】**

```
清晨六点，你检查着车上的气瓶。
今天有18单要送，3家欠瓶要收，
还有那个总说"下次一起算"的王老板...

以前，你带着笔记本、计算器、
和一颗怕算错账的心。

现在，只需要这个App。
```

**【场景二：路上的从容】**

```
电话响了，是李大妈："小张啊，今天送气吗？"

你一边导航，一边看她的欠瓶记录：
"送！您欠的那瓶15kg的，今天也给您带过去。"

账在心里，清清楚楚。

不用翻本子，不用靠记忆。
```

**【场景三：收工时的踏实】**

```
晚上八点，最后一单完成。
你点开"今日收入"——¥1,680。

系统自动算好了：
- 气款收入：¥1,320
- 收回来2个欠瓶：¥300
- 支出：进货¥860
- 今天净赚：¥820

终于不用再对着笔记本算半小时了。
```

**【品牌标语】**

```
LPG配送助手
让每一单都算得清，让每一天都赚得明。
```

## 2. 登录方式

### 2.1 支持的登录方式

| 登录方式 | 优先级 | 使用场景 | 说明 |
|---------|-------|---------|------|
| **手机号+验证码** | P0 | 主要登录方式 | 最常用，无需记忆密码 |
| **手机号+密码** | P1 | 二次登录 | 用户主动设置的密码 |
| **微信一键登录** | P1 | 快捷登录 | 微信授权，自动注册 |
| **自动登录** | P0 | 已登录用户 | Token续期，无感知 |

### 2.2 登录方式对比

| 维度 | 手机号+验证码 | 手机号+密码 | 微信登录 |
|-----|-------------|-----------|---------|
| 首次登录 | ✅ 支持 | ✅ 支持 | ✅ 支持 |
| 二次登录 | ✅ 支持 | ✅ 支持 | ✅ 支持 |
| 注册流程 | 自动注册 | 需设置密码 | 自动注册 |
| 安全性 | 中（验证码时效） | 高（密码强度） | 高（微信安全） |
| 便捷性 | 高 | 中（需记密码） | 高 |
| 适用场景 | 所有用户 | 高频用户 | 微信用户 |

## 3. 登录流程

### 3.1 登录流程（叙事版）

```
【第一章：初见】
打开App
    ↓
三张故事卡片缓缓展开（3秒/张）
    ├─ 清晨装车的忙碌
    ├─ 路上导航的从容  
    └─ 晚上对账的踏实
    ↓
"开始配送之旅"按钮亮起
    ↓

【第二章：相识】
检查本地Token
    ↓
Token有效 ──→ "欢迎回来，今天也要加油！" ──→ 进入首页
    ↓
Token无效/过期 ──→ 显示登录页
    ↓
"今天也要顺顺利利，多多赚钱"
    ↓
输入手机号
    ↓
点击"获取验证码" ──→ "验证码正在飞来，请稍候..."
    ↓
输入6位验证码
    ↓
点击"登录"
    ↓

【第三章：同行】
验证成功
    ├─ 新用户 ──→ 
    │   "告诉我你是谁，让我更好地帮你记账"
    │   进入完善信息页
    │   填写姓名、企业、区域
    │   "欢迎加入，今天就开始赚钱吧！"
    │   进入首页
    │
    └─ 老用户 ──→ 
        时段化问候（早上/中午/晚上）
        或天气关怀（雨天/高温天）
        进入首页
```

### 3.2 验证码规则

| 规则项 | 配置值 | 说明 |
|-------|-------|------|
| 验证码长度 | 6位数字 | 便于输入 |
| 有效期 | 5分钟 | 安全与便利平衡 |
| 获取间隔 | 60秒 | 防止刷短信 |
| 每日上限 | 10条/手机号 | 成本控制 |
| 错误次数 | 5次 | 超过需重新获取 |
| 验证码格式 | 纯数字 | 方便老年用户 |

### 3.3 Token机制

```
登录成功
    ↓
下发两个Token：
├─ AccessToken（访问令牌）
│   有效期：2小时
│   用途：API请求认证
│   存储：内存（安全）
│
└─ RefreshToken（刷新令牌）
    有效期：30天
    用途：刷新AccessToken
    存储：本地加密存储

使用流程：
1. 每次API请求携带AccessToken
2. AccessToken过期 → 用RefreshToken刷新
3. RefreshToken过期 → 需要重新登录
```

## 4. 注册流程

### 4.1 自动注册机制

**核心理念**：手机号即账户，首次登录自动注册，无需单独注册流程。

```
首次登录（手机号+验证码）
    ↓
系统检测手机号未注册
    ↓
自动创建用户账户
    ├─ 生成user_id
    ├─ 记录手机号
    ├─ 生成默认昵称："配送员138****5678"
    ├─ 生成默认头像（系统随机头像）
    ├─ 设置默认配置
    └─ 初始化数据
    ↓
可选择进入完善信息页，或直接进首页
    ↓
完成注册，进入首页
```

**简化设计**：
- 姓名：非必填，默认显示"配送员138****5678"
- 头像：非必填，系统随机分配默认头像
- 企业和区域：完全选填
- 用户可随时在"我的-个人资料"中修改

**优势**：
- 降低注册门槛，3秒内完成登录
- 不强制收集信息，尊重用户意愿
- 核心功能（开单、配送）不受信息完整度影响

## 6. 安全机制

### 6.1 异常登录检测

| 异常场景 | 检测规则 | 处理方式 |
|---------|---------|---------|
| 异地登录 | 非常用城市 | 短信提醒+需要验证码确认 |
| 新设备 | 首次出现的设备 | 短信提醒+设备绑定确认 |
| 频繁登录 | 1小时内登录5次以上 | 图形验证码+短信限速 |
| 密码错误 | 连续5次错误 | 账户锁定30分钟 |

### 6.2 登录保护

```
异常登录检测触发
    ↓
发送安全提醒（短信+推送）
    ↓
用户确认：
├─ 是我本人 ──→ 标记为可信设备
└─ 非本人 ──→ 强制下线+修改密码
```

## 7. 退出登录

### 7.1 退出方式

| 方式 | 触发条件 | 效果 |
|-----|---------|------|
| 主动退出 | 用户点击"退出登录" | 清除Token，返回登录页 |
| Token过期 | RefreshToken过期 | 提示重新登录 |
| 强制下线 | 安全策略/管理后台 | 立即退出，提示原因 |

### 7.2 退出流程

```
用户点击"退出登录"
    ↓
确认弹窗："确定退出登录？"
    ↓
确认
    ↓
清除本地Token
    ↓
清除本地敏感数据（保留缓存）
    ↓
通知服务端注销会话
    ↓
返回登录页
```

## 9. 情感化设计原则

### 9.1 为什么要用叙事风格

配送员是体力劳动者，每天面对：
- 体力劳动的疲惫
- 账目计算的压力
- 客户欠款的焦虑
- 风吹日晒的辛苦

**叙事风格的价值**：
1. **建立情感连接** - 让他们感到被理解，不是冰冷的工具
2. **降低心理门槛** - 温暖的开场，减少对新技术的抗拒
3. **正向激励** - 每一天的问候，都是小小的鼓励
4. **品牌温度** - 不只是App，是"懂你的配送伙伴"

### 9.2 叙事设计的三个层次

| 层次 | 目标 | 示例 |
|-----|------|------|
| **场景共鸣** | 让用户看到自己 | "清晨六点，你检查着车上的气瓶" |
| **痛点认同** | 让用户感到被理解 | "以前带着笔记本和计算器" |
| **解决方案** | 给用户希望 | "现在，只需要这个App" |

### 9.3 文案写作原则

**1. 用"你"，不用"用户"**
- ❌ "用户打开App"
- ✅ "你打开App"

**2. 具体，不抽象**
- ❌ "提高工作效率"
- ✅ "每天省下半小时对账时间"

**3. 有画面感**
- ❌ "快速开单"
- ✅ "3秒开单，比写个字还快"

**4. 正向表达**
- ❌ "不要错过"
- ✅ "记得开启，不会漏单"

**5. 适当口语化**
- ❌ "请输入正确的手机号格式"
- ✅ "手机号好像少了几位，再检查一下？"

### 9.4 时段化问候库

**早晨（6:00-9:00）**
- "早起的人儿赚钱多，今天也要元气满满！"
- "新的一天，新的订单，冲！"
- "早上好，检查一下今天的待送清单吧"

**中午（11:00-13:00）**
- "忙了一上午，记得吃午饭哦~"
- "中午好，补充能量，下午继续"
- "吃了吗？别饿着肚子送气"

**下午（14:00-18:00）**
- "下午好，今天目标还差X单，加油！"
- "下午容易犯困，注意安全"
- "再坚持一下，很快就收工了"

**晚上（18:00-22:00）**
- "晚上好，辛苦一天了，最后几单！"
- "天黑了，注意安全，慢慢开"
- "今天送了X单，真厉害！"

**特殊天气**
- 雨天："下雨天路滑，安全第一，不着急"
- 高温："今天X度，注意防暑，多喝水"
- 寒冷："今天X度，注意保暖，别感冒了"
- 大风："风大，气瓶固定好，注意安全"

### 9.5 登录成功后的正向反馈

**连续登录激励**
- 3天："连续3天打卡，真棒！"
- 7天："一周全勤，敬业！"
- 30天："一个月全勤，你是配送达人！"

**业绩里程碑**
- 今日首单："今天第一单，开门红！"
- 突破10单："今天已经10单了，效率真高！"
- 收工提醒："今天送了X单，辛苦了，早点休息"
