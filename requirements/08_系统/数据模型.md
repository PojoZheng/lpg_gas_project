# 系统模块数据模型

## 1. 实体关系图

```
┌───────────────┐     ┌───────────────┐     ┌───────────────┐
│     User      │     │   DataSyncLog │     │ Notification  │
│    用户账号    │◄────│   数据同步日志  │     │    通知消息    │
└───────────────┘     └───────────────┘     └───────────────┘
         │
         │ 1:N
         ▼
┌───────────────┐     ┌───────────────┐
│  LoginDevice  │     │ OfflineQueue  │
│   登录设备     │     │   离线队列     │
└───────────────┘     └───────────────┘
```

---

## 2. User 用户账号

### 2.1 表结构

| 字段名 | 类型 | 必填 | 默认值 | 说明 |
|-------|------|------|--------|------|
| id | STRING(32) | 是 | UUID | 主键 |
| phone | STRING(20) | 是 | - | 手机号，唯一 |
| name | STRING(50) | 否 | NULL | 姓名 |
| avatar | STRING(500) | 否 | NULL | 头像URL |
| company_id | STRING(32) | 否 | NULL | 所属企业ID |
| company_name | STRING(100) | 否 | NULL | 企业名称 |
| region_code | STRING(20) | 否 | NULL | 服务区域编码 |
| region_name | STRING(50) | 否 | NULL | 服务区域名称 |
| status | ENUM | 是 | ACTIVE | 账号状态 |
| last_login_at | DATETIME | 否 | NULL | 最后登录时间 |
| last_login_ip | STRING(50) | 否 | NULL | 最后登录IP |
| created_at | DATETIME | 是 | CURRENT | 创建时间 |
| updated_at | DATETIME | 是 | CURRENT | 更新时间 |

### 2.2 枚举定义

**账号状态 (status)**：
```
ACTIVE          = '正常'
INACTIVE        = '未激活'
SUSPENDED       = '已停用'
DELETED         = '已删除'
```

## 3. LoginDevice 登录设备

### 3.1 表结构

| 字段名 | 类型 | 必填 | 默认值 | 说明 |
|-------|------|------|--------|------|
| id | STRING(32) | 是 | UUID | 主键 |
| user_id | STRING(32) | 是 | - | 用户ID |
| device_id | STRING(100) | 是 | - | 设备唯一标识 |
| device_name | STRING(100) | 否 | NULL | 设备名称 |
| device_type | ENUM | 是 | - | 设备类型 |
| os_version | STRING(50) | 否 | NULL | 系统版本 |
| app_version | STRING(20) | 否 | NULL | App版本 |
| push_token | STRING(200) | 否 | NULL | 推送令牌 |
| is_current | BOOLEAN | 是 | FALSE | 是否当前设备 |
| last_login_at | DATETIME | 是 | CURRENT | 最后登录时间 |
| created_at | DATETIME | 是 | CURRENT | 创建时间 |

### 3.2 枚举定义

**设备类型 (device_type)**：
```
IOS             = 'iOS'
ANDROID         = 'Android'
OTHER           = '其他'
```

## 4. DataSyncLog 数据同步日志

### 4.1 表结构

| 字段名 | 类型 | 必填 | 默认值 | 说明 |
|-------|------|------|--------|------|
| id | STRING(32) | 是 | UUID | 主键 |
| user_id | STRING(32) | 是 | - | 用户ID |
| device_id | STRING(32) | 是 | - | 设备ID |
| sync_type | ENUM | 是 | - | 同步类型 |
| sync_direction | ENUM | 是 | - | 同步方向 |
| entity_type | STRING(50) | 是 | - | 实体类型 |
| entity_id | STRING(32) | 是 | - | 实体ID |
| operation | ENUM | 是 | - | 操作类型 |
| local_data | JSON | 否 | NULL | 本地数据 |
| server_data | JSON | 否 | NULL | 服务端数据 |
| resolved_data | JSON | 否 | NULL | 解决后的数据 |
| conflict_type | ENUM | 否 | NULL | 冲突类型 |
| status | ENUM | 是 | - | 同步状态 |
| error_msg | STRING(500) | 否 | NULL | 错误信息 |
| retry_count | INTEGER | 是 | 0 | 重试次数 |
| created_at | DATETIME | 是 | CURRENT | 创建时间 |
| completed_at | DATETIME | 否 | NULL | 完成时间 |

### 4.2 枚举定义

**同步类型 (sync_type)**：
```
AUTO            = '自动同步'
MANUAL          = '手动同步'
REALTIME        = '实时同步'
BATCH           = '批量同步'
```

**同步方向 (sync_direction)**：
```
UPLOAD          = '上传'
DOWNLOAD        = '下载'
BIDIRECTIONAL   = '双向'
```

**操作类型 (operation)**：
```
CREATE          = '创建'
UPDATE          = '更新'
DELETE          = '删除'
```

**冲突类型 (conflict_type)**：
```
VERSION         = '版本冲突'
DATA            = '数据冲突'
DELETED         = '已删除冲突'
```

**同步状态 (status)**：
```
PENDING         = '待同步'
SYNCING         = '同步中'
SUCCESS         = '成功'
FAILED          = '失败'
CONFLICT        = '冲突'
RESOLVED        = '已解决'
```

---

## 5. OfflineQueue 离线操作队列

### 5.1 表结构

| 字段名 | 类型 | 必填 | 默认值 | 说明 |
|-------|------|------|--------|------|
| id | STRING(32) | 是 | UUID | 主键 |
| user_id | STRING(32) | 是 | - | 用户ID |
| device_id | STRING(32) | 是 | - | 设备ID |
| operation_type | ENUM | 是 | - | 操作类型 |
| entity_type | STRING(50) | 是 | - | 实体类型 |
| entity_id | STRING(32) | 是 | - | 实体ID（本地临时ID） |
| server_entity_id | STRING(32) | 否 | NULL | 服务端实体ID |
| request_data | JSON | 是 | - | 请求数据 |
| priority | INTEGER | 是 | 5 | 优先级（1-10） |
| status | ENUM | 是 | PENDING | 状态 |
| error_count | INTEGER | 是 | 0 | 错误次数 |
| error_msg | STRING(500) | 否 | NULL | 错误信息 |
| created_at | DATETIME | 是 | CURRENT | 创建时间 |
| executed_at | DATETIME | 否 | NULL | 执行时间 |
| completed_at | DATETIME | 否 | NULL | 完成时间 |

### 5.2 枚举定义

**操作类型 (operation_type)**：
```
CREATE_ORDER    = '创建订单'
UPDATE_ORDER    = '更新订单'
COMPLETE_ORDER  = '完成订单'
CREATE_CUSTOMER = '创建客户'
UPDATE_CUSTOMER = '更新客户'
CREATE_INSPECTION = '创建安检'
UPDATE_INVENTORY = '更新库存'
```

**优先级 (priority)**：
```
1 = 最高（订单状态变更）
5 = 默认（常规操作）
10 = 最低（非关键操作）
```

**状态 (status)**：
```
PENDING         = '待执行'
PROCESSING      = '执行中'
SUCCESS         = '成功'
FAILED          = '失败'
CANCELLED       = '已取消'
```

## 6. Notification 通知消息

### 6.1 表结构

| 字段名 | 类型 | 必填 | 默认值 | 说明 |
|-------|------|------|--------|------|
| id | STRING(32) | 是 | UUID | 主键 |
| user_id | STRING(32) | 是 | - | 用户ID |
| notification_type | ENUM | 是 | - | 通知类型 |
| title | STRING(100) | 是 | - | 标题 |
| content | STRING(500) | 是 | - | 内容 |
| data | JSON | 否 | NULL | 附加数据 |
| priority | ENUM | 是 | NORMAL | 优先级 |
| is_read | BOOLEAN | 是 | FALSE | 是否已读 |
| read_at | DATETIME | 否 | NULL | 阅读时间 |
| action_type | STRING(50) | 否 | NULL | 点击动作类型 |
| action_data | JSON | 否 | NULL | 点击动作数据 |
| expire_at | DATETIME | 否 | NULL | 过期时间 |
| created_at | DATETIME | 是 | CURRENT | 创建时间 |

### 6.2 枚举定义

**通知类型 (notification_type)**：
```
INVENTORY_WARNING = '库存预警'
OWED_BOTTLE_WARNING = '欠瓶提醒'
OWED_MONEY_WARNING = '欠款提醒'
SETTLEMENT_REMIND = '对账提醒'
INSPECTION_REMIND = '安检提醒'
SYSTEM_NOTICE     = '系统通知'
ORDER_UPDATE      = '订单更新'
```

**优先级 (priority)**：
```
HIGH            = '高'
NORMAL          = '中'
LOW             = '低'
```

## 7. DataBackup 数据备份

### 7.1 表结构

| 字段名 | 类型 | 必填 | 默认值 | 说明 |
|-------|------|------|--------|------|
| id | STRING(32) | 是 | UUID | 主键 |
| user_id | STRING(32) | 是 | - | 用户ID |
| device_id | STRING(32) | 是 | - | 设备ID |
| backup_type | ENUM | 是 | - | 备份类型 |
| backup_scope | JSON | 是 | - | 备份范围 |
| file_size | INTEGER | 是 | - | 文件大小(字节) |
| file_url | STRING(500) | 否 | NULL | 文件URL |
| record_count | JSON | 是 | - | 各表记录数 |
| checksum | STRING(64) | 是 | - | 文件校验和 |
| status | ENUM | 是 | - | 状态 |
| error_msg | STRING(500) | 否 | NULL | 错误信息 |
| created_at | DATETIME | 是 | CURRENT | 创建时间 |
| completed_at | DATETIME | 否 | NULL | 完成时间 |

### 7.2 枚举定义

**备份类型 (backup_type)**：
```
AUTO            = '自动备份'
MANUAL          = '手动备份'
DEVICE_CHANGE   = '换机备份'
```

**备份范围 (backup_scope)**：
```json
{
  "customer": true,
  "order": true,
  "inventory": true,
  "finance": true,
  "inspection": true,
  "config": true
}
```

**状态 (status)**：
```
PENDING         = '待备份'
BACKING_UP      = '备份中'
SUCCESS         = '成功'
FAILED          = '失败'
```

## 8. 索引设计

| 表名 | 索引名 | 字段 | 类型 |
|-----|-------|------|------|
| user | idx_phone | phone | 唯一索引 |
| login_device | idx_user_device | user_id, device_id | 唯一索引 |
| login_device | idx_push_token | push_token | 普通索引 |
| data_sync_log | idx_user_time | user_id, created_at | 普通索引 |
| data_sync_log | idx_status | user_id, status | 普通索引 |
| offline_queue | idx_user_status | user_id, status, priority | 普通索引 |
| offline_queue | idx_entity | entity_type, entity_id | 普通索引 |
| notification | idx_user_read | user_id, is_read, created_at | 普通索引 |
| notification | idx_type | user_id, notification_type | 普通索引 |
| data_backup | idx_user_time | user_id, created_at | 普通索引 |
