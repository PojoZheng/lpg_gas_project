## 2. Customer 客户档案

### 2.1 表结构

| 字段名 | 类型 | 必填 | 默认值 | 说明 |
|-------|------|------|--------|------|
| id | STRING(32) | 是 | UUID | 主键 |
| dealer_id | STRING(32) | 是 | - | 配送员ID |
| name | STRING(50) | 是 | - | 客户姓名 |
| phone | STRING(20) | 是 | - | 手机号，唯一索引 |
| type | ENUM | 是 | RESIDENTIAL | 客户类型 |
| address | STRING(200) | 是 | - | 详细地址 |
| longitude | DECIMAL(10,7) | 否 | NULL | 经度 |
| latitude | DECIMAL(10,7) | 否 | NULL | 纬度 |
| delivery_preference | ENUM | 否 | ANYTIME | 配送偏好 |
| tags | JSON | 否 | [] | 标签数组 |
| remark | STRING(500) | 否 | - | 备注 |
| status | ENUM | 是 | ACTIVE | 客户状态 |
| total_orders | INTEGER | 是 | 0 | 累计订单数 |
| last_order_at | DATETIME | 否 | NULL | 最后下单时间 |
| created_at | DATETIME | 是 | CURRENT | 创建时间 |
| updated_at | DATETIME | 是 | CURRENT | 更新时间 |

### 2.2 枚举定义

**客户类型 (type)**：
```
RESIDENTIAL   = '住宅'
SHOP          = '商铺'
RESTAURANT    = '餐馆'
FACTORY       = '工厂'
OTHER         = '其他'
```

**配送偏好 (delivery_preference)**：
```
MORNING       = '上午'
AFTERNOON     = '下午'
ANYTIME       = '随时'
```

**客户状态 (status)**：
```
ACTIVE        = '正常'
WARNING       = '预警'
INACTIVE      = '停用'
DELETED       = '已删除'
```

## 3. DepositAccount 押金租金账户

### 3.1 表结构

| 字段名 | 类型 | 必填 | 默认值 | 说明 |
|-------|------|------|--------|------|
| id | STRING(32) | 是 | UUID | 主键 |
| customer_id | STRING(32) | 是 | - | 客户ID |
| dealer_id | STRING(32) | 是 | - | 配送员ID |
| spec_15kg_count | INTEGER | 是 | 0 | 15kg瓶数 |
| spec_10kg_count | INTEGER | 是 | 0 | 10kg瓶数 |
| spec_50kg_count | INTEGER | 是 | 0 | 50kg瓶数 |
| monthly_rent_count | INTEGER | 是 | 0 | 月租瓶数 |
| yearly_rent_count | INTEGER | 是 | 0 | 年租瓶数 |
| next_rent_date | DATE | 否 | NULL | 下次收租日期 |
| total_deposit_amount | DECIMAL(10,2) | 是 | 0 | 押金总额 |
| total_rent_amount | DECIMAL(10,2) | 是 | 0 | 租金总额 |
| created_at | DATETIME | 是 | CURRENT | 创建时间 |
| updated_at | DATETIME | 是 | CURRENT | 更新时间 |

### 3.2 计算逻辑

**押金总额计算**：
```javascript
total_deposit_amount = 
  spec_15kg_count * config.deposit_15kg +
  spec_10kg_count * config.deposit_10kg +
  spec_50kg_count * config.deposit_50kg
```

**租金总额计算**：
```javascript
total_rent_amount = 
  monthly_rent_count * config.rent_monthly +
  yearly_rent_count * config.rent_yearly
```

## 7. 视图/查询

### 7.1 客户列表视图

```sql
-- 客户列表查询（含统计信息）
SELECT 
  c.*,
  da.total_deposit_amount,
  da.total_rent_amount,
  COUNT(DISTINCT ow.id) as owed_count,
  SUM(CASE WHEN ow.status = 'OWING' THEN ow.owed_count ELSE 0 END) as total_owed_bottles,
  MAX(o.created_at) as last_order_time
FROM customer c
LEFT JOIN deposit_account da ON c.id = da.customer_id
LEFT JOIN owed_record ow ON c.id = ow.customer_id
LEFT JOIN `order` o ON c.id = o.customer_id
WHERE c.dealer_id = ? AND c.status != 'DELETED'
GROUP BY c.id
ORDER BY last_order_time DESC
```

### 7.2 客户预警视图

```sql
-- 客户预警查询
SELECT 
  c.*,
  COUNT(DISTINCT ow.id) as owed_bottle_count,
  DATEDIFF(CURRENT_DATE, ow.owed_date) as owed_days,
  DATEDIFF(CURRENT_DATE, c.last_order_at) as no_order_days
FROM customer c
LEFT JOIN owed_record ow ON c.id = ow.customer_id AND ow.status = 'OWING'
WHERE c.dealer_id = ?
  AND (
    (ow.id IS NOT NULL AND DATEDIFF(CURRENT_DATE, ow.owed_date) > ow.remind_days)
    OR DATEDIFF(CURRENT_DATE, c.last_order_at) > 30
  )
GROUP BY c.id
```

## 9. 数据约束

### 9.1 业务约束

| 约束 | 说明 |
|-----|------|
| 手机号唯一 | 同一配送员下客户手机号唯一 |
| 押金非负 | 各规格押金瓶数不能为负 |
| 欠瓶状态一致 | 还瓶数量不能超过欠瓶数量 |
| 级联更新 | 客户删除时关联记录标记删除 |

### 9.2 触发器

```sql
-- 客户最后下单时间自动更新
CREATE TRIGGER update_customer_last_order
AFTER INSERT ON `order`
FOR EACH ROW
BEGIN
  UPDATE customer 
  SET last_order_at = NEW.created_at,
      total_orders = total_orders + 1
  WHERE id = NEW.customer_id;
END;
```
