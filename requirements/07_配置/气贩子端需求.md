# 气贩子端配置需求

> 关联文档：[01_平台端配置需求.md](./01_平台端配置需求.md)

---

## 1. 需求概述

气贩子端配置功能下放给气贩子自主设定，涉及**业务规则、市场策略、客户关系**等，气贩子可根据自己的经营策略灵活调整。

---

## 2. 配置模块需求

### 2.1 押金租金配置

**需求描述**：气贩子需设置各规格气瓶的押金和租金标准，以及按客户的差异化收取规则

#### 2.1.1 押金金额设置

| 配置项 | 类型 | 说明 |
|-------|------|------|
| 15kg气瓶押金 | number | 如150元 |
| 10kg气瓶押金 | number | 如100元 |
| 50kg气瓶押金 | number | 如400元 |

#### 2.1.2 租金金额设置

| 配置项 | 类型 | 说明 |
|-------|------|------|
| 月租金 | number | 如5元/月 |
| 年租金 | number | 如50元/年 |

#### 2.1.3 客户差异化规则

**需求描述**：气贩子可为不同客户设置不同的押金租金收取策略

| 策略类型 | 说明 |
|---------|------|
| 收取 | 正常收取押金和租金 |
| 免收 | 老客户/大客户免收 |
| 押金内扣 | 租金从押金中按月扣除 |

**配置示例**：
- 李大妈（老客户）：免押金、免租金
- 王老板（新客户）：收押金150元、收租金5元/月
- 张餐馆（大客户）：免押金、租金从押金扣

---

### 2.2 残液处理配置

**需求描述**：气贩子需设置残液单价和默认处理方式

**配置项**：
| 配置项 | 类型 | 说明 |
|-------|------|------|
| 功能开关 | boolean | 是否启用残液记录功能 |
| 残液单价 | number | 元/kg，气贩子自主定价（如5元/kg） |
| 默认处理方式 | enum | 抵扣气款/单独收入/忽略不计 |

**处理方式说明**：
| 方式 | 说明 |
|-----|------|
| 抵扣气款 | 残液价值抵扣当次气款，实收金额减少 |
| 单独收入 | 残液单独记录为收入，气款全额收取 |
| 忽略不计 | 不记录残液，视为利润空间 |

---

### 2.3 欠瓶管理配置

**需求描述**：气贩子需设置欠瓶风险控制的规则参数

**配置项**：
| 配置项 | 类型 | 说明 |
|-------|------|------|
| 功能开关 | boolean | 是否启用欠瓶跟踪 |
| 提醒天数 | array | 欠瓶后第几天提醒（如[7, 15, 30]） |
| 最长欠瓶天数 | number | 如60天 |
| 超期处理方式 | enum | 转为收入/继续欠着 |

**超期处理方式**：
| 方式 | 说明 |
|-----|------|
| 转为收入 | 用客户押金抵扣欠瓶转为收入；无押金则转为客户欠款 |
| 继续欠着 | 继续跟踪，由气贩子自行处理 |

---

### 2.4 欠款管理配置

**需求描述**：气贩子需设置客户信用额度和欠款风险控制

**配置项**：
| 配置项 | 类型 | 说明 |
|-------|------|------|
| 功能开关 | boolean | 是否启用欠款管理 |
| 最大欠款额度 | number | 单个客户最多可欠金额（如1000元） |
| 最长欠款天数 | number | 如30天 |
| 提醒天数 | array | 欠款后第几天提醒（如[7, 15, 25]） |

---

### 2.5 库存预警配置

**需求描述**：气贩子需设置库存安全线和预警参数

**配置项**：
| 配置项 | 类型 | 说明 |
|-------|------|------|
| 预警开关 | boolean | 是否启用库存预警 |
| 重瓶安全库存 | object | 各规格重瓶最低库存（如15kg低于5瓶提醒） |
| 空瓶安全库存 | object | 各规格空瓶最低库存 |
| 过期预警天数 | number | 气瓶到期前多少天提醒（如30天） |

---

## 3. 配置界面需求

### 3.1 配置入口
- 在气贩子App的"我的"页面设置"业务规则"入口

### 3.2 配置页面结构
```
业务规则设置
├── 押金租金设置
│   ├── 押金标准
│   ├── 租金标准
│   └── 客户特殊规则
├── 残液处理设置
├── 欠瓶管理设置
├── 欠款管理设置
└── 库存预警设置
```

### 3.3 配置交互需求

**通用需求**：
- 配置项使用表单控件（输入框、开关、选择器）
- 数值项支持数字输入，带单位提示
- 修改后需点击"保存"才生效
- 支持恢复默认设置

**客户差异化规则**：
- 支持从客户列表选择客户
- 支持批量设置客户规则
- 直观显示每个客户的当前规则

### 3.4 配置生效
- 配置保存后立即生效
- 新订单按新规则执行
- 历史订单不受影响

---

## 4. 配置数据说明

### 4.1 初始值
- 新注册气贩子使用系统默认值
- 默认值基于行业平均水平

### 4.2 数据存储
- 配置数据存储在服务端
- 气贩子更换设备可同步配置
