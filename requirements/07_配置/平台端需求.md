# 平台端配置需求

> 关联文档：[02_气贩子端配置需求.md](./02_气贩子端配置需求.md)

---

## 1. 需求概述

平台端配置功能用于管理涉及**法规、监管、溯源**要求的配置项，由平台运营人员统一管理，气贩子不可修改。

---

## 2. 配置模块需求

### 2.1 地区模板配置

**需求描述**：平台需支持按地区设置不同的合规模板

**配置项**：
| 配置项 | 类型 | 可选值 | 说明 |
|-------|------|-------|------|
| 地区名称 | string | - | 如"广东省深圳市" |
| 合规模板 | enum | strict/standard/loose | 严格/标准/宽松模式 |

**模板定义**：
| 模板 | 适用场景 |
|-----|---------|
| strict（严格）| 一线城市，安检要求高 |
| standard（标准）| 二线城市 |
| loose（宽松）| 三四线城市 |

---

### 2.2 气瓶扫码配置

**需求描述**：平台需配置气瓶扫码溯源功能的启用策略

**配置项**：
| 配置项 | 类型 | 说明 |
|-------|------|------|
| 功能开关 | boolean | 是否开启气瓶扫码功能 |
| 是否强制 | boolean | 是否强制要求扫码（否表示建议但不强制） |
| 扫码方式 | enum | qr_code（二维码）/ rfid（RFID）/ manual（手动录入） |
| 扫码场景 | array | 配送时/回收时/充装时 |

---

### 2.3 安检规则配置

**需求描述**：平台需配置安检流程的规则和参数

#### 2.3.1 随瓶安检配置

| 配置项 | 类型 | 说明 |
|-------|------|------|
| 功能开关 | boolean | 是否开启随瓶安检 |
| 安检方式 | enum | ai_assist（AI辅助）/ photo_only（仅拍照）/ manual（人工） |
| AI识别项 | array | 气瓶检测/软管检测/灶具检测/泄漏检测 |
| 必拍项目 | array | 气瓶照片/环境照片 |

#### 2.3.2 定期安检配置

| 配置项 | 类型 | 说明 |
|-------|------|------|
| 功能开关 | boolean | 是否开启定期安检任务 |
| 检查周期 | number | 间隔天数（如90天） |
| 自动任务 | boolean | 是否自动生成安检任务 |

---

### 2.4 数据上报配置

**需求描述**：平台需配置数据上报给政府和燃企的方式

**配置项**：
| 配置项 | 类型 | 说明 |
|-------|------|------|
| 上报模式 | enum | company_first（燃企中转）/ direct（直连政府）/ hybrid（混合） |
| 批量间隔 | string | 批量上报时间间隔（如24h） |

**三种上报模式**：

**模式A：燃企中转（company_first）**
- 数据流向：气贩子 → 燃企 → 政府
- 气贩子数据先上报给燃企，燃企审核后再上报政府
- 支持批量上报（非实时）

**模式B：直连政府（direct）**
- 数据流向：气贩子 → 政府
- 气贩子数据直接上报政府
- 支持实时上报

**模式C：混合模式（hybrid）**
- 常规数据：气贩子 → 燃企（批量）
- 隐患数据：气贩子 → 政府（实时）

---

## 3. 配置管理功能需求

### 3.1 配置列表
- 支持查看所有配置项的当前值
- 支持按模块筛选（扫码/安检/上报）
- 支持搜索配置项

### 3.2 配置编辑
- 支持修改配置值
- 修改前需二次确认
- 记录修改日志（修改人、修改时间、旧值、新值）

### 3.3 配置生效
- 配置修改后实时生效
- 气贩子端自动同步最新配置

### 3.4 配置版本
- 记录配置变更历史
- 支持查看历史版本
- 支持回滚到历史版本
