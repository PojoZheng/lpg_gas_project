# 订单模块需求文档

## 1. 模块定位

**一句话描述**：管理从开单到完成的全流程，支持订单查询、状态跟踪、配送导航、完成收款。

**与首页的关系**：
- 首页是"任务看板"（只看今天、只看待办）
- 订单模块是"完整档案"（看全部、看历史、管理详情）

---

## 2. 入口与页面结构

### 2.1 入口来源

| 入口 | 来源 | 进入页面 | 默认状态 |
|-----|------|---------|---------|
| "查看全部待送" | 首页-下一个配送区 | 订单列表 | 筛选"待配送" |
| "我的-订单历史" | 我的页面 | 订单列表 | 全部订单 |
| 客户详情-历史订单 | 客户模块 | 订单列表 | 筛选该客户 |
| 首页快速开单完成 | 首页 | 订单列表（可选跳转） | 刷新显示新订单 |

### 2.2 页面结构

```
订单模块
├── 订单列表页（主页面）
│   ├── 顶部筛选栏
│   ├── 订单列表
│   └── 批量操作（可选）
├── 订单详情页
│   ├── 订单信息
│   ├── 操作按钮（根据状态）
│   └── 修改记录
└── 完成订单页（当场交易）
    ├── 收款确认
    ├── 金额修改
    └── 生成凭证
```

### 4.2 订单状态与可操作动作

| 状态 | 可操作 | 不可操作 |
|-----|--------|---------|
| 待配送 | 送达并收款、修改订单、取消订单 | 退款、评价 |

| 已退货 | 查看原因、再次开单 | 配送、修改 |

### 4.3 修改订单功能

**可修改项（24小时内）**：
| 字段 | 修改限制 | 影响 |
|-----|---------|------|
| 规格数量 | 可修改，但需库存充足 | 库存联动调整 |
| 单价 | 可修改 | 应收金额重新计算 |
| 实收金额 | 可修改 | 收入统计调整 |
| 收款方式 | 可修改 | 财务记录调整 |
| 押金/租金 | 可增删 | 客户账户调整 |

**修改记录**：
- 每次修改生成一条修改记录
- 记录修改人（自己）、修改时间、修改前后值
- 显示在订单详情底部"修改记录"区

### 4.4 取消订单流程

```
点击[取消订单]

"确定取消此订单？取消后库存将释放。"
[确认取消] [再想想]
    ↓
选择取消原因（可选）：
- 客户取消
- 库存不足
- 价格问题
- 其他：____
    ↓
订单状态变为"已取消"
    ↓
库存回滚（如果有锁定）
    ↓
如果已收款，触发退款流程
```

## 6. 欠瓶处理功能

### 6.1 什么是欠瓶

**业务场景**：客户购买液化气（重瓶）时，手头没有空瓶可以交换，配送员允许客户"欠"一个空瓶，下次配送时收回。

**为什么要记录**：
- 防止忘记收瓶
- 统计客户欠瓶情况
- 超期提醒催收
- 可转为收入（客户买瓶）

### 6.2 欠瓶产生流程

```
订单完成时（点击"送达并收款"）
    ↓
弹出"完成订单"面板
    ↓
系统询问："客户有15kg空瓶交换吗？"
    ↓
┌─────────────────────────────────────┐
│ 选项A：[✅ 有] 正常收回空瓶          │
│ 选项B：[❌ 没有-现在收现金] 收押金   │
│ 选项C：[🔄 欠着] 下次再收           │
└─────────────────────────────────────┘
    ↓
选择"欠着"
    ↓
填写欠瓶信息：
- 规格：15kg（默认与配送规格相同）
- 数量：1瓶
- 处理方式：下次收 / 转为收入 / 免除
    ↓
生成欠瓶记录
    ↓
订单完成，提示"已记录欠瓶1瓶"
```

### 6.3 欠瓶处理选项

| 选项 | 适用场景 | 后续处理 |
|-----|---------|---------|
| **下次配送时收** | 客户确实没有空瓶，下次再给 | 生成欠瓶记录，下次配送提醒 |
| **转为收入** | 客户不想还瓶，按市场价买下 | 欠瓶转为销售收入，更新客户押金账户 |
| **免除** | 配送员决定送客户一个瓶 | 欠瓶记录标记为免除，不追讨 |

### 6.4 欠瓶提醒与催收

**提醒机制**：
| 时机 | 方式 | 内容 |
|-----|------|------|
| 产生欠瓶时 | Toast | "已记录欠瓶1瓶，下次配送提醒" |
| 每日首页 | 徽章/提示 | "2位客户欠瓶待收" |
| 配送该客户时 | 订单标记 | 订单卡片显示"⚠️ 欠1瓶" |
| 超期7天 | 推送通知 | "李大妈欠瓶已超7天未还" |
| 超期15天 | 推送+首页标红 | "建议转为收入" |
| 超期30天 | 按配置自动处理 | 自动转为收入或继续欠 |

**收回欠瓶流程**：
```
下次配送该客户
    ↓
订单完成面板显示"该客户欠1瓶15kg"
    ↓
收回空瓶时选择"收回欠瓶"
    ↓
欠瓶记录更新为"已收回"
    ↓
客户欠瓶清零
```

### 6.5 欠瓶数据

**欠瓶记录字段**：
| 字段 | 说明 |
|-----|------|
| 客户信息 | 欠瓶客户 |
| 规格数量 | 欠瓶规格和数量 |
| 欠瓶日期 | 产生欠瓶的日期 |
| 欠瓶天数 | 自动计算，用于提醒 |
| 处理方式 | 下次收/转收入/免除 |
| 关联订单 | 产生欠瓶的订单ID |
| 状态 | 欠瓶中/已收回/已转收入/已免除 |

**统计维度**：
- 当前欠瓶总数（按规格）
- 欠瓶超期客户数
- 欠瓶转为收入金额

## 7. 批量开单功能

### 7.1 使用场景

早上开工时，复制昨天的订单快速生成今天的订单。

### 7.2 流程

```
订单列表页 → [批量开单]
    ↓
选择日期（默认昨天）
    ↓
显示该日期所有订单列表
    ↓
勾选要复制的订单（可多选）
    ↓
点击[确认生成]
    ↓
批量创建订单，状态为"待配送"
    ↓
提示"成功创建X个订单"
    ↓
刷新订单列表
```

### 7.3 复制规则

| 字段 | 复制规则 | 说明 |
|-----|---------|------|
| 客户信息 | 完全复制 | 客户ID、地址 |
| 规格数量 | 完全复制 | 保持相同 |
| 单价 | 使用最新价格 | 如果有价格调整，用新价格 |
| 押金/租金 | 根据客户当前规则重新计算 | 客户可能已交押金 |
| 订单日期 | 更新为今天 | 新订单日期 |
| 订单状态 | 待配送 | 默认待配送 |

## 8. 与首页的联动

### 8.1 数据同步

| 场景 | 首页更新 | 订单模块更新 |
|-----|---------|-------------|
| 在首页快速开单 | 立即显示新订单 | 订单列表刷新后显示 |
| 在订单页完成配送 | 首页待送数-1，已收更新 | 当前页面更新 |
| 修改订单金额 | 首页收入重新计算 | 当前订单更新 |
| 取消订单 | 首页待送数-1 | 订单状态变更 |

### 8.2 状态一致性

- 首页显示"今日待送3家"，订单列表筛选"待配送"也应该是3家
- 数据以服务器为准，本地缓存用于快速展示
- 进入订单页时强制刷新，确保数据最新
