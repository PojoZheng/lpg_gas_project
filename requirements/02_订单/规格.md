### 1.2 订单列表手势交互

| 手势 | 作用对象 | 效果 | 参数 |
|-----|---------|------|------|
| 点击 | 订单项 | 进入订单详情 | 整行可点击 |
| 左滑 | 待配送订单 | 显示"删除"按钮 | 触发距离>80px，最大120px |
| 右滑 | 待配送订单 | 显示"标记完成"按钮 | 同上 |
| 下拉 | 列表顶部 | 刷新数据 | 触发距离80px，loading高度60px |
| 上滑 | 列表底部 | 加载更多 | 距离底部20px触发 |
| 长按 | 订单项 | 显示快捷操作菜单 | 触发时间500ms |

**重要设计原则**：
- 没有"配送中"状态：气贩子配送过程不需要在App中标记状态
- 只有"待配送"（还没送）和"已完成"（已经送完并收款）
- 送达时直接点击"送达并收款"完成订单
- 没有配送费：气贩子自己配送，配送成本不计入订单金额

**左滑删除流程**：
```
用户左滑订单项
    ↓
显示红色"删除"按钮（宽度80px）
    ↓
点击删除
    ↓
弹出确认对话框："确定删除此订单？删除后不可恢复"
    ↓
确认 → 删除订单，Toast"已删除"
    ↓
取消 → 订单项回弹，恢复原始状态
```

### 1.3 Tab切换交互

```
点击[待配送] Tab
    ↓
当前列表淡出（150ms）
    ↓
加载待配送订单数据
    ↓
新列表淡入（200ms）
    ↓
Tab下划线滑动动画（200ms，ease-out）
```

**切换保持状态**：
- 切换Tab后，列表滚动位置重置到顶部
- 搜索关键词清空
- 但返回后再进入，记住上次筛选状态

## 欠瓶处理流程（订单完成时）

### 欠瓶场景说明

**什么是欠瓶**：客户买了新气瓶（重瓶），但当时没有空瓶可以交换，所以"欠"一个空瓶，下次配送时收回。

**触发时机**：点击"送达并收款"时，系统检查客户是否有空瓶交换

### 欠瓶处理交互流程

```
点击"送达并收款"
    ↓
弹出"完成订单"面板
    ↓
┌─────────────────────────────────────────┐
│  完成订单 - 李大妈                       │
├─────────────────────────────────────────┤
│  配送：15kg × 1瓶    ¥120               │
│                                         │
│  ──── 空瓶收回 ────                     │
│                                         │
│  客户有15kg空瓶吗？                      │
│                                         │
│  ┌────────┐  ┌────────┐  ┌────────┐    │
│  │ ✅ 有   │  │ ❌ 没有 │  │ 🔄 欠着 │    │
│  │ 正常收  │  │ 现在收  │  │ 下次收  │    │
│  └────────┘  └────────┘  └────────┘    │
│                                         │
│  【选择"欠着"时展开】                    │
│  ┌─────────────────────────────────┐    │
│  │ 欠瓶规格：15kg × 1瓶             │    │
│  │                                 │    │
│  │ 欠瓶处理方式：                   │    │
│  │ ● 下次配送时收（默认）          │    │
│  │ ○ 转为收入（按¥120收）          │    │
│  │ ○ 免除（送客户了）              │    │
│  │                                 │    │
│  │ 预计还瓶时间：7天内              │    │
│  │ 到期提醒：7/15/30天             │    │
│  └─────────────────────────────────┘    │
│                                         │
│  ──── 收款 ────                         │
│                                         │
│  应收：¥120    [微信收款] [现金] [记账]  │
│                                         │
│     [     完 成 订 单     ]             │
└─────────────────────────────────────────┘
```

### 欠瓶处理选项说明

| 选项 | 使用场景 | 后续处理 |
|-----|---------|---------|
| **有 - 正常收** | 客户有空瓶 | 正常完成订单，库存：重瓶出库，空瓶入库 |
| **没有 - 现在收** | 客户暂时没空瓶，但可以现在给现金买瓶 | 收现金作为押金，订单完成 |
| **欠着 - 下次收** | 客户确实没有，下次再收 | 生成欠瓶记录，提醒配送员 |

### 欠瓶状态流转

```
订单完成时选择"欠着"
    ↓
生成欠瓶记录（状态：欠瓶中）
    ↓
首页显示"有客户欠瓶"提醒
    ↓
下次给该客户配送时
    ↓
首页/订单列表显示"⚠️ 该客户欠1瓶"
    ↓
送达时弹出"收回欠瓶"选项
    ↓
┌─────────────────────────────────────┐
│ 收回欠瓶                             │
│ 客户：李大妈                         │
│ 欠瓶：15kg × 1瓶（欠于04-01）       │
│                                     │
│ [✓ 已收回空瓶]  [转为收入]  [继续欠] │
└─────────────────────────────────────┘
    ↓
选择处理方式
    ↓
欠瓶记录更新（已收回/已转收入/继续欠）
```

### 欠瓶提醒机制

| 提醒时机 | 提醒方式 | 内容 |
|---------|---------|------|
| 生成欠瓶时 | Toast | "已记录欠瓶1瓶，下次配送提醒" |
| 每日首页 | 徽章/文字 | "2位客户欠瓶待收" |
| 配送该客户时 | 订单标记 | 订单卡片显示"⚠️ 欠1瓶" |
| 超期7天 | 推送通知 | "李大妈欠瓶已超7天未还" |
| 超期15天 | 推送+首页 | 红色提醒，建议转为收入 |
| 超期30天 | 自动处理 | 按配置自动转为收入或继续欠 |

### 欠瓶数据记录

欠瓶记录包含：
- 客户信息
- 欠瓶规格、数量
- 欠瓶日期
- 预计还瓶时间
- 处理方式
- 关联订单

欠瓶收回时：
- 更新欠瓶记录状态
- 可选：生成新的收入记录（如转为收入）
- 更新客户欠瓶统计

**状态卡片动画**：
- 进入页面时，状态标签从左侧滑入（300ms）
- 主按钮从底部弹入，带弹性效果（400ms，spring）

## 3. 订单状态机

### 3.1 状态流转图

```
┌─────────┐    创建     ┌─────────┐    点击送达    ┌─────────┐
│  草稿   │ ──────────→ │ 待配送  │ ───────────→ │ 已完成  │
│ (可选)  │             │         │    并收款      │         │
└─────────┘             └────┬────┘              └────┬────┘
                             │                        │
                    ┌────────┴────────┐               │
                    │                 │               │
                    ▼                 ▼               │
              ┌─────────┐      ┌─────────┐          │
              │ 已取消  │      │ 修改订单 │          │
              │         │      │ (24h内) │          │
              └─────────┘      └────┬────┘          │
                                    │               │
                                    └───────────────┤
                                                    │
              ┌─────────────────────────────────────┘
              │
              │ 快捷换货（5分钟内）
              ▼
        ┌─────────┐
        │ 换货中  │
        │ (中间态)│
        └────┬────┘
              │
              │ 换货完成
              ▼
        ┌─────────┐
        │ 已完成  │
        │ (新订单)│
        └─────────┘
```

### 3.2 状态定义

| 状态 | 编码 | 说明 | 可执行操作 |
|-----|------|------|-----------|
| draft | 草稿 | 临时保存，未确认 | 编辑、确认、删除 |
| pending | 待配送 | 已确认，等待配送 | 点击送达、修改、取消 |
| completed | 已完成 | 已送达并收款 | 再次开单、修改（24h内）、退货 |
| cancelled | 已取消 | 订单已取消 | 再次开单 |
| returning | 退货中 | 退货申请处理中 | - |
| returned | 已退货 | 退货已完成 | 再次开单 |

### 3.3 状态变更记录

每次状态变更记录日志：
```
OrderStatusLog
- id: 日志ID
- order_id: 订单ID
- from_status: 原状态
- to_status: 新状态
- operator: 操作人（dealer_id）
- remark: 备注
- created_at: 变更时间
```

## 4. 边界条件

### 4.1 修改订单边界

| 场景 | 限制 | 处理方式 |
|-----|------|---------|
| 修改时间 | 完成后24小时内 | 超过24小时隐藏[修改]按钮 |
| 修改次数 | 无限制 | 但保留所有修改记录 |
| 规格修改 | 需库存充足 | 库存不足时提示并禁止修改 |
| 金额修改 | 任意金额 | 生成修改记录，收入统计调整 |
| 已取消订单 | 不可修改 | [修改]按钮禁用 |

### 4.2 删除订单边界

| 场景 | 能否删除 | 处理方式 |
|-----|---------|---------|
| 待配送订单 | ✅ 可以 | 确认对话框，删除后库存回滚 |
| 已完成订单 | ❌ 不可以 | 只能退货，不能删除 |
| 已完成订单 | ❌ 不可以 | 只能退货，不能删除 |
| 已取消订单 | ✅ 可以 | 物理删除或标记删除 |

### 4.3 列表加载边界

| 场景 | 处理方式 |
|-----|---------|
| 无订单 | 显示空状态插图+文案"还没有订单，点击开单" |
| 网络错误 | 显示缓存数据+顶部提示"离线模式" |
| 加载失败 | 显示"加载失败"+重试按钮 |
| 数据量大 | 分页加载，每页20条 |
| 搜索无结果 | "未找到相关订单，换个关键词试试" |

## 6. 与首页的协作

### 6.1 数据同步机制

```
首页和订单列表的数据同步：

1. 首页显示"今日待送3家"
   ↓（来自本地缓存）
2. 点击"查看全部待送"
   ↓
3. 进入订单列表，强制刷新
   ↓
4. 获取最新数据（可能是4家）
   ↓
5. 订单列表显示4家
   ↓
6. 返回首页，首页自动刷新
   ↓
7. 首页显示"今日待送4家"（数字跳动更新）
```

### 6.2 状态一致性保障

| 场景 | 首页 | 订单列表 | 处理方式 |
|-----|------|---------|---------|
| 在订单列表完成配送 | 待送-1 | 状态变已完成 | 返回首页时刷新 |
| 在首页取消订单 | 待送-1 | 需要同步 | 进入订单列表时刷新 |
| 在订单列表修改金额 | 收入不变 | 金额更新 | 返回首页时刷新，收入重新计算 |

### 6.3 跳转参数

```
首页 → 订单列表：
{
  "filter": "pending",      // 默认筛选状态
  "date": "today"           // 日期筛选
}

我的 → 订单列表：
{
  "filter": "all",          // 全部订单
  "date": null              // 不限日期
}

客户详情 → 订单列表：
{
  "customer_id": "C001",    // 客户筛选
  "filter": "all"
}
```

## 7. 错误处理

### 7.1 操作错误

| 错误 | 提示方式 | 文案 |
|-----|---------|------|
| 完成订单失败 | Toast | "完成订单失败，请重试" |
| 取消订单失败 | Dialog | "订单取消失败，可能已配送" |
| 修改订单超时 | Toast | "网络超时，修改未保存" |
| 删除订单失败 | Toast | "删除失败，请稍后重试" |

### 7.2 数据错误

| 错误 | 处理方式 |
|-----|---------|
| 订单不存在 | 返回列表页，Toast"订单不存在或已删除" |
| 订单状态异常 | 刷新订单，显示最新状态 |
| 数据加载失败 | 显示缓存，提示"离线数据，可能不是最新" |

## 8. 性能规格

### 8.1 加载性能

| 指标 | 目标 | 优化方案 |
|-----|------|---------|
| 列表首屏加载 | < 1s | 本地缓存+骨架屏 |
| 列表刷新 | < 1.5s | 增量更新 |
| 详情页加载 | < 800ms | 本地缓存优先 |
| Tab切换 | < 300ms | 数据预加载 |

### 8.2 列表性能

| 指标 | 目标 | 方案 |
|-----|------|------|
| 长列表滚动 | > 50fps | 虚拟列表（>100条时） |
| 图片加载 | 懒加载 | 进入视口才加载 |
| 内存占用 | < 50MB | 页面离开时释放资源 |
