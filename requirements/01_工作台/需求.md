# 工作台（首页）需求文档

## 1. 页面定位

**一句话描述**：配送员每天打开App的第一屏，展示今日收入、待配送任务、快速开单入口。

**用户价值**：
- 3秒内知道今天赚了多少钱
- 5秒内知道今天还有多少家要送
- 10秒内完成快速开单

## 3. 模块详细设计

### 3.1 今日收入卡（区域A）

#### 3.1.1 字段定义

| 字段名 | 类型 | 计算逻辑 | 显示规则 |
|-------|------|---------|---------|
| 今日已收 | 金额 | 今日所有订单实收金额之和 | 大字体，粗体，绿色 |
| 今日待收 | 金额 | 今日所有订单应收未收金额（欠款） | 中字体，灰色 |
| 今日毛利 | 金额 | 今日收入 - 今日成本 | 小字体，展开后显示 |
| 本周趋势 | 数组 | 近7天每日已收金额 | 折线图，点击展开 |

#### 3.1.2 计算逻辑

```
今日已收 = SUM(今日订单.实收金额)
         WHERE 订单日期 = 今日 AND 收款状态 = 已收款

今日待收 = SUM(今日订单.应收金额 - 实收金额)
         WHERE 订单日期 = 今日 AND 收款状态 = 未收完

今日成本 = SUM(今日进货成本)
         （配送成本不计入，配送员自己送）

今日毛利 = 今日已收 - 今日成本
```

### 3.2 下一个配送（区域B）

#### 3.2.1 排序规则

待配送订单默认排序：
1. **有预约时间**的按预约时间升序（先约的先送）
2. **无预约时间**的按创建时间升序（先开的先送）
3. **同时间**的按距离升序（近的优先）

#### 3.2.2 客户卡片字段

| 字段名 | 类型 | 说明 | 显示规则 |
|-------|------|------|---------|
| 客户姓名 | 字符串 | 客户档案中的姓名 | 大字体，左对齐 |
| 客户标签 | 数组 | VIP/大客户/免押金等 | 小标签，姓名右侧 |
| 配送地址 | 字符串 | 订单配送地址 | 单行，超出省略 |
| 气瓶规格 | 字符串 | 15kg/10kg/50kg | 图标+文字 |
| 气瓶数量 | 数字 | 配送数量 | 数字+"瓶" |
| 应收金额 | 金额 | 气款+押金+租金 | 粗体，右对齐 |
| 预约时间 | 时间 | 客户预约的配送时间 | 如果有则显示，红色提醒 |
| 欠瓶数量 | 数字 | 该客户当前欠瓶数 | 如果有则显示，黄色提醒 |
| 欠款金额 | 金额 | 该客户当前欠款金额 | 如果有则显示，红色提醒 |

## 4. 快速开单流程

### 4.1 标准流程

```
点击[快速开单]
    ↓
底部弹出"选择客户"面板
    ↓
选择客户 → 弹出"确认订单"面板
    ↓
确认规格/数量/金额
    ↓
选择订单类型：
    ├─ [稍后配送] → 创建待配送订单 → 加入待送列表
    └─ [当场完成] → 弹出"完成订单"面板 → 确认收款 → 订单直接完成
```

### 4.2 订单类型说明

| 类型 | 使用场景 | 库存处理 | 收入统计 |
|-----|---------|---------|---------|
| **稍后配送** | 需要送货上门的订单 | 立即锁定库存，待配送状态 | 配送完成后计入收入 |
| **当场完成** | 当场给气当场收钱的订单 | 立即扣减库存，已完成状态 | 立即计入今日已收 |

### 4.3 库存处理逻辑（关键）

#### 稍后配送订单

```
确认开单时：
1. 立即锁定库存（重瓶-1，状态：锁定待配送）
2. 订单状态：待配送
3. 送达并收款后：
   - 重瓶正式消耗（状态：已售出）
   - 空瓶+1（收回空瓶时）
4. 如果取消订单：
   - 库存回滚（锁定→可售）
```

#### 当场完成订单

```
确认开单时：
1. 立即扣减库存（重瓶-1，状态：已售出）
2. 订单状态：已完成
3. 同时弹出收款面板
4. 确认收款后：
   - 计入今日已收
   - 如果有收回空瓶，空瓶+1
```

## 6. 订单修改与撤销

### 6.1 5秒撤销机制

```
完成交易后，Toast提示：
"✓ 交易完成 ¥130  [撤销]"  （5秒倒计时进度条）

点击撤销后：
→ 订单回退到编辑状态（不是删除）
→ 库存回滚
→ 收入扣除
→ 显示"已撤销，可修改"
```

### 6.2 24小时内修改规格数量

```
订单详情页 → [修改订单]

可修改项：
- 实收金额（抹零补差场景）
- 规格数量（报错修正）
- 备注信息

修改限制：
- 24小时内可修改
- 修改后生成修改记录
- 原记录保留（对账可查）
- 涉及库存变更的，同步调整库存
```

### 6.3 修改库存联动规则

```
原订单：15kg×1
修改后：15kg×2（客户又要了一瓶）

处理：
1. 检查库存是否充足
2. 库存-1（再扣一瓶）
3. 应收金额增加
4. 记录为订单修改，不是新订单
```

## 7. 离线场景处理

### 7.1 本地队列机制

```
网络中断时：
1. 点击完成 → 本地先记录交易
2. Toast提示："交易已记录，网络恢复后自动同步"
3. 首页收入数字立即更新（本地计算，乐观显示）
4. 顶部显示"离线模式，X笔交易待同步"
5. 网络恢复后后台自动同步
6. 同步冲突时（如库存不足），标记为"异常订单"需人工处理
```

### 7.2 冲突处理

| 冲突场景 | 处理方式 |
|---------|---------|
| 本地库存够，服务器库存不够 | 标记异常，提示"库存不足，请检查" |
| 本地完成，服务器显示已取消 | 以本地为准，强制同步 |
| 金额不一致 | 以本地为准，生成对账差异记录 |

## 8. 数据一致性保障

### 8.1 首页与"我的"页面一致性

```
保障机制：
1. 首页收入数据从本地数据库读取（实时）
2. 每次进入"我的"页面，强制同步服务器数据
3. 如果发现差异（服务器 ≠ 本地）：
   - 以服务器数据为准
   - 本地数据刷新
   - 提示"数据已更新"
4. 提供"刷新"按钮，手动同步
```

### 8.2 事务机制（账户操作）

```
一个订单可能涉及多个账户操作：
1. 气款收入 +¥130
2. 客户存款 +¥500（充值）
3. 客户存款 -¥130（抵扣）
4. 押金收入 +¥150（新客）

事务保证：
- 前端：显示"处理中..."，按钮禁用，防止重复点击
- 后端：使用数据库事务，全部成功或全部回滚
- 失败处理：任何一步失败，提示"交易失败，请重试"，无数据变更
```

## 10. 约束规则

### 10.1 数据加载规则

| 场景 | 规则 |
|-----|------|
| 页面首次加载 | 显示缓存数据，后台异步刷新 |
| 下拉刷新 | 强制刷新今日收入、待配送列表 |
| 自动刷新 | 每5分钟自动刷新一次（后台静默） |
| 无网络 | 显示缓存数据，顶部提示"离线模式，X笔待同步" |

### 10.2 空状态处理

| 场景 | 显示内容 |
|-----|---------|
| 今日无订单 | "今天还没有订单，点击快速开单开始吧！" |
| 今日订单都已完成 | "今日任务全部完成！已收¥X" |
| 无待配送订单 | "待配送订单为空" |
| 网络错误 | "网络出小差了，下拉重试" |
| 离线待同步 | "离线模式，X笔交易待同步" |

### 10.3 性能约束

| 指标 | 要求 |
|-----|------|
| 页面加载时间 | < 1秒（缓存数据） |
| 数据刷新时间 | < 2秒 |
| 点击响应时间 | < 100ms |
| 面板弹出时间 | < 200ms |

## 12. 退货/换货逆向流程

### 12.1 使用场景

**场景A：当场发现气瓶问题**
```
当场完成订单后，客户立即发现气瓶有问题（阀门漏气、瓶身损坏）
需要当场退货或换货
```

**场景B：已完成的订单后续退货**
```
昨天送的订单，今天客户发现气瓶有问题，要求退货
```

### 12.2 当场换货流程（交易完成后5分钟内）

```
交易完成后，页面显示[快捷换货]按钮（5分钟内有效）

点击[快捷换货]：
1. 原订单标记为"换货中"
2. 库存回滚（坏瓶回到库存，标记为待检）
3. 弹出新的开单面板，规格默认相同
4. 新开一单，完成交付
5. 系统记录为"换货"，关联原订单
```

| 操作 | 订单状态 | 库存影响 | 收入影响 | 客户账户 |
|-----|---------|---------|---------|---------|
| 当场换货 | 原：换货中 → 新：已完成 | 原瓶回库（待检），新瓶出库 | 按差价调整 | 不变 |
| 24小时内退货 | 原：已退货 | 重瓶回库（待检） | 今日已收扣除 | 如有存款，优先退存款 |

### 12.5 异常处理

**气瓶已使用**：
- 如果是质量问题，全额退款，气瓶报废处理
- 如果是客户原因，协商退款金额（如扣除部分气款）

**气瓶丢失/客户不还**：
- 标记为"气瓶未收回"
- 从客户押金扣除（如有押金）
- 或转为欠瓶记录
