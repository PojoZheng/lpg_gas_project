│   Cylinder    │◄────│InventoryLog   │
│   气瓶档案     │     │ 库存变动日志   │
└───────────────┘     └───────┬───────┘

          ┌───────────────────┼───────────────────┐

## 2. Cylinder 气瓶档案

### 2.1 表结构

| 字段名 | 类型 | 必填 | 默认值 | 说明 |
|-------|------|------|--------|------|
| id | STRING(32) | 是 | UUID | 主键 |
| dealer_id | STRING(32) | 是 | - | 配送员ID |
| cylinder_no | STRING(50) | 是 | - | 气瓶编号，唯一 |
| spec | STRING(10) | 是 | - | 规格：15kg/10kg/50kg |
| status | ENUM | 是 | FULL | 气瓶状态 |
| location | ENUM | 是 | WAREHOUSE | 当前位置 |
| current_order_id | STRING(32) | 否 | NULL | 当前关联订单ID |
| customer_id | STRING(32) | 否 | NULL | 当前客户ID |
| manufacture_date | DATE | 否 | NULL | 生产日期 |
| last_check_date | DATE | 否 | NULL | 上次检验日期 |
| next_check_date | DATE | 否 | NULL | 下次检验日期 |
| check_status | ENUM | 是 | VALID | 检验状态 |
| purchase_date | DATE | 否 | NULL | 采购日期 |
| purchase_price | DECIMAL(10,2) | 否 | NULL | 采购价格 |
| total_delivery_count | INTEGER | 是 | 0 | 累计配送次数 |
| last_delivery_date | DATE | 否 | NULL | 最后配送日期 |
| created_at | DATETIME | 是 | CURRENT | 创建时间 |
| updated_at | DATETIME | 是 | CURRENT | 更新时间 |

### 2.2 枚举定义

**气瓶状态 (status)**：
```
FULL            = '重瓶'        // 满瓶可配送
EMPTY           = '空瓶'        // 空瓶待充装
LOCKED          = '锁定'        // 已被订单锁定
IN_DELIVERY     = '配送中'      // 已出库配送
SCRAPPED        = '已报废'      // 气瓶已报废
```

**当前位置 (location)**：
```
WAREHOUSE       = '库房'
DELIVERY        = '配送中'
CUSTOMER        = '客户处'
STATION         = '充装站'
SCRAP_YARD      = '报废区'
```

**检验状态 (check_status)**：
```
VALID           = '合格'
NEAR_EXPIRE     = '即将过期'    // 30天内到期
EXPIRED         = '已过期'
CHECKING        = '检验中'
```

## 3. InventoryLog 库存变动日志

### 3.1 表结构

| 字段名 | 类型 | 必填 | 默认值 | 说明 |
|-------|------|------|--------|------|
| id | STRING(32) | 是 | UUID | 主键 |
| dealer_id | STRING(32) | 是 | - | 配送员ID |
| cylinder_id | STRING(32) | 否 | NULL | 气瓶ID（单瓶追踪时） |
| change_type | ENUM | 是 | - | 变动类型 |
| spec | STRING(10) | 是 | - | 规格 |
| quantity | INTEGER | 是 | - | 变动数量（正增负减） |
| before_count | INTEGER | 是 | - | 变动前数量 |
| after_count | INTEGER | 是 | - | 变动后数量 |
| order_id | STRING(32) | 否 | NULL | 关联订单ID |
| customer_id | STRING(32) | 否 | NULL | 关联客户ID |
| purchase_id | STRING(32) | 否 | NULL | 关联采购记录ID |
| refill_id | STRING(32) | 否 | NULL | 关联充装记录ID |
| cylinder_ids | JSON | 否 | NULL | 气瓶ID列表（扫码追踪时）|
| remark | STRING(200) | 否 | - | 备注 |
| created_at | DATETIME | 是 | CURRENT | 变动时间 |

### 3.2 枚举定义

**变动类型 (change_type)**：
```
// 入库类
PURCHASE_IN     = '采购入库'    // 采购重瓶入库 +
FILL_IN         = '充装入库'    // 充装后重瓶入库 +
RETURN_IN       = '退瓶入库'    // 回收空瓶入库 +
TRANSFER_IN     = '调拨入库'    // 调拨重瓶入库 +

// 出库类
SEND_FILL       = '送充装'      // 空瓶送充装出库 -  [新增]
DELIVERY_OUT    = '配送出库'    // 配送重瓶出库 -
TRANSFER_OUT    = '调拨出库'    // 调拨出库 -
SCRAP_OUT       = '报废出库'    // 报废出库 -

// 状态变更类
LOCK            = '订单锁定'    // 可用→锁定
UNLOCK          = '订单解锁'    // 锁定→可用
COMPLETE        = '订单完成'    // 锁定→出库
```

## 4. InventorySnapshot 库存快照

### 4.1 表结构

| 字段名 | 类型 | 必填 | 默认值 | 说明 |
|-------|------|------|--------|------|
| id | STRING(32) | 是 | UUID | 主键 |
| dealer_id | STRING(32) | 是 | - | 配送员ID |
| spec | STRING(10) | 是 | - | 规格 |
| full_count | INTEGER | 是 | 0 | 重瓶数量 |
| empty_count | INTEGER | 是 | 0 | 空瓶数量 |
| locked_count | INTEGER | 是 | 0 | 锁定数量 |
| available_count | INTEGER | 是 | 0 | 可用数量 = 重瓶-锁定 |
| warning_line | INTEGER | 是 | 5 | 预警线 |
| snapshot_date | DATE | 是 | CURRENT | 快照日期 |
| created_at | DATETIME | 是 | CURRENT | 创建时间 |

### 4.2 计算字段

```javascript
// 可用库存
available_count = full_count - locked_count

// 库存状态
if (available_count <= 0) {
  status = 'OUT_OF_STOCK'     // 缺货
} else if (available_count <= warning_line) {
  status = 'LOW_STOCK'        // 紧张
} else {
  status = 'NORMAL'           // 正常
}
```

## 5. RefillRecord 充装记录

### 5.1 表结构

| 字段名 | 类型 | 必填 | 默认值 | 说明 |
|-------|------|------|--------|------|
| id | STRING(32) | 是 | UUID | 主键，自动生成 |
| dealer_id | STRING(32) | 是 | - | 配送员ID |
| refill_station | STRING(100) | 否 | - | 充装站名称 |
| refill_date | DATE | 是 | CURRENT | 充装日期 |
| spec | STRING(10) | 是 | - | 规格：15kg/10kg/50kg |
| empty_count | INTEGER | 是 | 0 | 送充空瓶数 |
| fill_count | INTEGER | 是 | 0 | 充回重瓶数 |
| gas_weight | DECIMAL(8,2) | 否 | NULL | 充装气体重量(kg) |
| unit_price | DECIMAL(10,2) | 否 | NULL | 充装单价(元/瓶或元/kg) |
| total_cost | DECIMAL(10,2) | 否 | NULL | 充装总费用(元) |
| remark | STRING(200) | 否 | - | 备注 |
| created_at | DATETIME | 是 | CURRENT | 创建时间 |

### 5.2 业务说明

**充装 vs 采购的区别**：

| 维度 | 充装 (Refill) | 采购 (Purchase) |
|-----|---------------|-----------------|
| 流程 | 空瓶→充装站→重瓶 | 直接购买重瓶 |
| 库存影响 | 空瓶-N，重瓶+N | 重瓶+N |
| 成本 | 充装费（不含瓶） | 采购价（含瓶） |
| 追踪 | 同一气瓶状态变更 | 新气瓶建档 |

**业务流程**：
1. 送空瓶到充装站，扫空瓶码出库（可选）
2. 取回重瓶，扫重瓶码入库（可选）
3. 记录充装费用和气体重量
4. 创建 RefillRecord，自动生成两条 InventoryLog（SEND_FILL + FILL_IN）

## 8. 视图/查询

### 7.1 库存总览查询

```sql
-- 配送员库存总览
SELECT 
  spec,
  SUM(CASE WHEN status = 'FULL' THEN 1 ELSE 0 END) as full_count,
  SUM(CASE WHEN status = 'EMPTY' THEN 1 ELSE 0 END) as empty_count,
  SUM(CASE WHEN status = 'LOCKED' THEN 1 ELSE 0 END) as locked_count,
  SUM(CASE WHEN status = 'FULL' AND location = 'WAREHOUSE' THEN 1 ELSE 0 END) as available_count
FROM cylinder
WHERE dealer_id = ? AND status != 'SCRAPPED'
GROUP BY spec
```

### 7.2 超期锁定查询

```sql
-- 超期锁定的气瓶（锁定超过3天）
SELECT 
  c.*,
  o.id as order_id,
  o.customer_id,
  o.created_at as lock_time,
  DATEDIFF(CURRENT_DATE, o.created_at) as lock_days
FROM cylinder c
JOIN `order` o ON c.current_order_id = o.id
WHERE c.dealer_id = ? 
  AND c.status = 'LOCKED'
  AND DATEDIFF(CURRENT_DATE, o.created_at) > 3
```

### 7.3 即将过期气瓶查询

```sql
-- 30天内到期的气瓶
SELECT *
FROM cylinder
WHERE dealer_id = ?
  AND next_check_date IS NOT NULL
  AND next_check_date <= DATE_ADD(CURRENT_DATE, INTERVAL 30 DAY)
  AND status != 'SCRAPPED'
ORDER BY next_check_date ASC
```
