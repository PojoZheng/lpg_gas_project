## 1. 产品定位

**一句话描述**：让配送员开单更快、算账更清、配送更顺、安检合规无忧的数字化助手。

**目标用户**：个体液化气配送员（配送员），日均配送20-100瓶

---

## 2. 核心价值（量化版）

### 2.1 效率提升

| 指标 | 传统方式 | 使用App后 | 节省 |
|-----|---------|----------|------|
| 开单时间 | 手写2分钟/单 | 15秒/单 | **节省87%** |
| 日结对账 | 30分钟/天 | 3分钟/天 | **节省90%** |
| 找客户地址 | 平均5分钟/单 | 地图导航2分钟/单 | **节省60%** |
| 算利润 | 算不清 | 实时查看 | **从模糊到清晰** |

**测算依据**：
- 开单：手写需记录客户、规格、数量、金额，App预选客户自动计算
- 对账：传统翻笔记本，App一键生成日结报表
- 找地址：凭记忆 vs 地图导航

### 2.2 风险控制

| 风险类型 | 传统方式 | App解决 |
|---------|---------|---------|
| 欠瓶丢失 | 口头约定，无记录 | 系统记录+自动提醒，回收率提升40% |
| 欠款超期 | 记不清谁欠多久 | 自动提醒+信用额度控制，回款周期缩短50% |
| 安检漏做 | 容易忘 | 配送强制关联，完成率>98% |

### 2.3 收入增长

- **减少欠瓶损失**：按日均3个欠瓶、回收率提升40%计算，月减少损失约¥540（3瓶×¥150×40%×30天）
- **加快回款**：回款周期从15天降到7天，资金周转更快
- **增值收入**：语音开单等高级功能，提升服务体验，可支撑溢价

## 3. 产品形态：数字档口

**不是"管理系统"，是"数字档口"**

配送员打开App，第一眼看到的是今天能赚多少钱。

### 3.1 首页信息架构（3-Tab设计）

**导航原则**：底部仅3个常驻Tab，其他功能通过"我的"页面或流程触发进入。

```
┌─────────────────────────────────────────┐
│  【今日收入卡】                          │  ← 核心数据区
│  今日已收  ¥1,680                        │
│  今日待收  ¥540                          │
│  [点击展开趋势图]                        │
├─────────────────────────────────────────┤
│  【下一个配送】                          │  ← 配送任务区
│  ┌─────────────────────────────────┐   │
│  │ 李大妈  15kg×1  ¥135            │   │
│  │ 阳光花园5栋                     │   │
│  │ [开始配送] [电话] [更多]        │   │
│  └─────────────────────────────────┘   │
│       [查看全部待送（3家）]              │
├─────────────────────────────────────────┤
│  【底部操作区】                          │  ← 主要操作区
│                                         │
│     [        ⚡ 快速开单        ]       │
│                                         │
│     🎤 语音开单（邀请3人免费30天）       │
│                                         │
├─────────────────────────────────────────┤
│  🏠首页      👥客户      👤我的          │  ← 底部导航（仅3个）
└─────────────────────────────────────────┘
```

**底部导航说明**：
| Tab | 核心功能 | 其他入口 |
|-----|---------|---------|
| **首页** | 收入展示、快速开单、下一个配送 | 展开趋势图、查看全部待送 |
| **客户** | 客户列表、客户档案 | 从客户页开单 |
| **我的** | 个人设置、库存管理、安检记录、财务对账 | 所有低频管理功能 |

> **设计原则**：
> - 首页聚焦「赚钱」和「送气」两个核心场景
> - 3秒内知道今天赚了多少钱，5秒内知道还有几家要送
> - 库存、安检等功能不单独设Tab，也不放首页入口，全部聚合到"我的"页面

### 3.2 信息可视化原则

1. **一眼可见**：核心数据（收入、待办）不滚动就能看到
2. **对比清晰**：已收/待收、收入/成本并排显示
3. **趋势直观**：本周收入折线图，判断淡旺季
4. **异常提醒**：欠瓶超期、库存不足用红色突出（点击跳转"我的-库存管理"）

## 4. 两端架构

| 端 | 定位 | 核心功能 | 用户 |
|---|------|---------|------|
| **配送员端** | 核心业务操作 | 开单、配送、库存、对账 | 配送员 |
| **平台运营端** | 监控+配置 | 配送员管理、数据监控、扫码规则配置 | 平台运营 |

### 4.1 登录认证

**极简登录流程**：
```
手机号 + 短信验证码
    ↓
首次登录自动注册
    ↓
生成默认昵称（配送员138****5678）和头像
    ↓
可选完善信息，或直接进首页
```

**特点**：
- 无独立注册流程，手机号即账户
- 姓名/头像/企业/区域完全选填
- 降低门槛，3秒内完成登录

## 8. 关键指标定义

### 8.1 日活跃（DAU）
- **定义**：当日登录App，并完成至少一个核心操作（开单/配送确认/收款）的用户
- **计算公式**：当日活跃用户数 / 累计注册用户数

### 8.2 开单时间
- **定义**：从点击"开单"按钮到提交订单的时长
- **测量点**：埋点记录操作时长

### 8.3 欠瓶回收率
- **定义**：30天内状态变为"已收回"的欠瓶占比
- **计算公式**：30天内回收的欠瓶数 / 30天前产生的欠瓶总数

## 9. 核心设计原则

### 9.1 导航极简
- 底部仅3个Tab：首页、客户、我的
- 库存、安检等功能不单独设Tab，也不放首页入口
- 低频功能全部聚合到"我的"页面

### 9.2 流程闭环
- **开单-配送-安检**：订单完成后自动触发安检
- **库存-充装**：充装流程一步完成（扫空瓶→填信息→扫重瓶）
- **欠瓶-提醒**：系统自动提醒，支持多轮催收

### 9.3 气瓶追踪
- 支持气瓶扫码，记录每个气瓶的全生命周期
- 平台可配置是否强制扫码（满足监管要求）
- 不扫码时支持数量级管理，降低使用门槛
