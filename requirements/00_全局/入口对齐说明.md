# 入口对齐说明

> 本文档说明各模块入口如何与首页设计对齐

---

## 核心原则

**首页设计确定的导航结构**：
```
┌─────────────────────────────────────────┐
│                                         │
│     🏠          👥          👤         │
│    首页        客户         我的        │
│                                         │
└─────────────────────────────────────────┘
         ↑
    只有这3个底部Tab！
```

**其他模块入口策略**：
1. **高频功能** → 首页快捷入口（工作台区域）
2. **中频功能** → 我的页面入口
3. **低频功能** → 收进二级页面或设置
4. **流程触发** → 自动弹出（如安检）

---

## 各模块入口对齐详情

### 1. 工作台（首页）

| 功能 | 入口位置 | 说明 |
|-----|---------|------|
| 待送列表 | 首页顶部卡片 | 最核心功能，一眼看到下一单 |
| 快速开单 | 首页中部大按钮 | 每天高频使用 |
| 收入概览 | 首页卡片 | 今日已收/待收，点击进入财务 |

**首页不放库存/安检列表入口**：
- 库存：首页无入口，库存管理从「我的-库存管理」进入
- 安检：首页无入口，订单完成后自动弹出

---

### 2. 客户模块

| 功能 | 入口位置 | 说明 |
|-----|---------|------|
| 客户列表 | **底部Tab-客户** | 独立Tab，方便查找 |
| 客户详情 | 客户列表点击 | 查看客户信息 |
| 新增客户 | 客户列表右上角+ | 快速添加 |
| 快捷开单 | 客户详情/列表 | 左滑或按钮 |

**对齐状态**：✅ 已有独立Tab，无需调整

---

### 3. 订单模块

| 功能 | 入口位置 | 说明 |
|-----|---------|------|
| 待送订单 | 首页待送列表 | 主要入口 |
| 订单历史 | 我的 → 订单历史 | 非高频，收进我的 |
| 订单详情 | 待送列表点击 | 查看/操作订单 |
| 创建订单 | 首页快速开单 | 主入口 |

**调整说明**：
- ❌ 删除：底部「订单」Tab（与首页重复）
- ✅ 保留：首页待送列表（更直接）
- ✅ 新增：我的 → 订单历史（查看已完成）

### 4. 库存模块

| 功能 | 原设计 | 调整后 | 入口位置 |
|-----|-------|-------|---------|
| 库存总览 | 独立Tab | ❌ 删除Tab | 我的-库存管理 |
| 入库操作 | 独立页面 | ✅ 保留 | 开单时库存不足提示 / 我的-库存管理 |
| 气瓶明细 | 独立页面 | ✅ 保留（低频） | 我的-库存管理-查看明细 |

**调整说明**：
- ❌ 删除：底部「库存」Tab
- ❌ 删除：首页库存快捷图标
- ✅ 保留：我的 → 库存管理（主要入口）

**产品经理说明**：
> 配送员不需要在首页查看库存，只在开单时如果库存不足会提示，或主动到「我的-库存管理」查看。

### 5. 安检模块

| 功能 | 原设计 | 调整后 | 入口位置 |
|-----|-------|-------|---------|
| 随瓶安检 | 独立Tab | ❌ 删除Tab | 订单完成后自动弹出（主入口） |
| 补录安检 | 独立页面 | ✅ 保留 | 首页长按订单 / 我的-安检记录 |
| 安检记录 | 独立页面 | ✅ 保留 | 我的 → 安检记录 |

**调整说明**：
- ❌ 删除：底部「安检」Tab
- ✅ 新增：订单完成后自动弹出安检页面（主入口）
- ✅ 保留：我的 → 安检记录（查看历史）

**产品经理说明**：
> 安检是「配送流程的附属动作」，不是独立功能。配送员不想多记一件事，必须跟着订单走。完成订单→自动弹出安检→10秒完成→结束。

## 配送员视角的入口记忆

```
每天工作流程：

1. 打开App → 自动在【首页】
   ↓
2. 看一眼【待送列表】，点击配送
   ↓
3. 送完 → 自动弹出【安检】→ 拍照完成
   ↓
4. 需要开单 → 首页【快速开单】
   ↓
5. 看一眼【库存状态表情】，不够就去进货
   ↓
6. 结束配送 → 看一眼【收入卡片】今天赚了多少
   ↓
7. 深度管理 → 【我的】页面找

不需要记：
- 安检在哪里？→ 送完自动弹出
- 库存去哪里看？→ 首页有表情，点表情进去
- 财务去哪里看？→ 首页有数字，点数字进去
```

## 冲突解决记录

### 冲突1：库存入口
- **首页设计**：首页显示库存状态表情（😊😐😰），点击预警进入补货
- **调整方案**：首页不进入库存列表，库存列表只能从「我的-库存管理」进入

### 冲突2：安检入口
- **首页设计**：首页无安检入口，订单完成后自动弹出
- **调整方案**：首页不放任何安检入口， purely 作为订单流程的一部分

### 冲突3：财务入口
- **首页设计**：首页收入卡片显示今日已收/待收
- **原财务规格**：可能有独立Tab
- **解决方案**：删除独立Tab，主入口改为「首页收入卡片点击」

### 冲突4：订单历史入口
- **首页设计**：首页只有「待送列表」，不显示已完成订单
- **原订单规格**：可能有订单Tab显示全部
- **解决方案**：订单历史收进「我的-订单历史」，首页只显示待配送

## 关键设计决策（以首页为准）

1. **底部Tab只有3个**：首页、客户、我的
2. **订单完成后自动弹出安检**：不是独立功能，是配送流程一部分
3. **首页收入卡片可点击**：点击进入财务详情
4. **首页显示库存表情**：😊😐😰 一眼看懂状态
5. **订单历史收进我的**：首页只显示待配送
6. **所有深度功能收进我的**：配置、统计、历史记录等
