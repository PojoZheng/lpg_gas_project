## 2. 关键业务场景

### 2.1 进货：采购 vs 充装

**采购（买新瓶）**：
```
场景：从供应商直接购买新瓶（已充装好的重瓶）
流程：
1. 进入"我的-库存管理"
2. 点击【采购入库】
3. 选择规格、输入数量
4. 扫描气瓶码（根据平台配置是否强制）
5. 输入单价、供应商（选填）
6. 确认入库

结果：
- 重瓶库存 +N
- 新气瓶建档（扫码时）
- 生成采购记录
- 记录采购成本（含瓶成本）
```

**充装（已有空瓶）**：
```
场景：将回收的空瓶送去充装站充装
流程：
1. 进入"我的-库存管理"
2. 点击【气瓶充装】
3. 【扫空瓶出库】扫描空瓶条码（可选）
4. 【填写充装信息】
   - 充装站、规格
   - 充装气量：75kg（5瓶×15kg）
   - 费用：¥425（¥85/瓶）
5. 【扫重瓶入库】扫描充装后的重瓶条码（可选）
6. 确认完成

结果：
- 空瓶库存 -5
- 重瓶库存 +5
- 生成充装记录
- 记录充装费用（不含瓶成本）
- 同一气瓶状态变更（扫码追踪时）
```

**对比**：
| 维度 | 采购 | 充装 |
|-----|------|------|
| 库存变化 | 重瓶+N | 空瓶-N，重瓶+N |
| 成本构成 | 含瓶成本 | 充装费（气+充装服务）|
| 气瓶来源 | 新气瓶建档 | 已有空瓶状态变更 |
| 适用场景 | 新增库存、新瓶补充 | 日常经营、空瓶周转 |

### 2.2 随瓶安检流程

**安检触发**：订单完成后自动触发，无需单独进入安检模块

```
配送完成
    ↓
自动弹出安检页面
    ↓
【必拍项】（根据平台配置）
- 气瓶整体照片
- 气瓶阀门照片
- 使用环境照片
    ↓
【AI识别】（可选）
- 识别气瓶外观异常
- 识别环境隐患
    ↓
【结果确认】
- 正常：提交完成
- 隐患：标记隐患类型，通知客户整改
    ↓
数据自动上报政府监管平台
```

**特点**：
- 不单独设安检入口，随配送流程完成
- 安检数据自动上报，满足合规要求
- 入口在"我的-安检记录"可查看历史

### 2.3 欠瓶管理

**欠瓶产生场景**：
```
场景A：新用户首次订气
- 用户没有空瓶可换
- 配送员送重瓶，未收回空瓶
- 产生1个"欠瓶"，系统记录并跟踪

场景B：用户空瓶丢失/损坏
- 用户说"空瓶找不到了"
- 配送员可选择：
  a) 让用户赔押金（¥150）
  b) 继续欠着，下次收回
  c) 从押金扣（如果已收押金）
  d) 不赔钱（自己承担损失，维系客户关系）
```

**欠瓶法律关系说明**：
- 欠瓶本质是客户欠配送员的气瓶资产
- 系统记录可作为双方确认的凭证
- **电子签名**：可作为配置项开启，增强法律效力

**系统支持**：
| 功能 | 说明 |
|-----|------|
| 欠瓶记录 | 自动记录每次欠瓶 |
| 欠瓶提醒 | 7/15/30天自动提醒（可配置） |
| 欠瓶转化 | 转为收入（用押金抵扣）、转为欠款、欠瓶作废 |
| 欠瓶统计 | 按时间维度统计欠瓶分布 |

### 2.4 残液处理

**残液来源**：用户退回的空瓶内有剩余液化气（约0.5-2kg/瓶）

**处理方式**：

| 方式 | 说明 | 适用场景 |
|-----|------|---------|
| **抵扣气款** | 残液价值抵扣当次气款 | 通用 |
| **单独收入** | 残液单独记录，气款全额收取 | 残液较多时 |
| **忽略不计** | 不记录，视为利润空间 | 小配送员简化操作 |

**示例**：
```
订单：15kg气瓶 × 1瓶，气款¥120，残液1kg×¥5=¥5
- 抵扣方式：实收¥115，残液抵扣¥5
- 单收方式：实收¥120，残液单独记录收入¥5
```

### 2.5 押金与租金

**三种典型场景**：

```
场景1：标准押金+租金
新用户：押金¥150/瓶（退瓶时退），租金¥5/月

场景2：押金内扣租金
用户已交押金¥150，每月租金¥5从押金扣，30个月后押金为0

场景3：市场化竞争（有的收有的不收）
李大妈（老客户）：免押金免租金
王老板（新客户）：收押金¥150 + 租金¥5/月
张餐馆（大客户）：免押金免租金
```

### 2.6 退换货流程

**退货场景**：客户收到气瓶后发现问题，要求退货

**退货流程**：
```
客户申请退货 
    ↓
配送员核实原因（质量问题/客户原因）
    ↓
┌─────────────────────────────────────┐
│ 情况A：质量问题                       │
│ - 配送员收回重瓶                     │
│ - 标记"质量问题"                     │
│ - 退款给客户（气款+配送费）           │
│ - 配送员向燃企索赔或自行承担          │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 情况B：客户原因（不想要了）            │
│ - 配送员收回重瓶（需完好）            │
│ - 退款给客户（气款，配送费不退）       │
│ - 重瓶回到库存                       │
└─────────────────────────────────────┘
    ↓
系统记录退货
    ↓
更新库存（重瓶+1）
    ↓
财务记录退款
```

**换货场景**：客户需要更换规格（如15kg换10kg）

**换货流程**：
```
客户申请换货
    ↓
收回原规格重瓶
    ↓
配送新规格重瓶
    ↓
结算差价（多退少补）
    ↓
系统记录换货
```

**退货/换货对数据的影响**：
- 订单状态变为"已退货"/"已换货"
- 库存相应调整
- 财务记录退款/补差
- 不计入正常收入统计

## 3. 异常流程

### 3.1 客户拒收

```
配送到达 → 客户拒收 → 标记拒收原因 → 退回库存 → 记录损耗
```

### 3.2 气瓶损坏

```
收回空瓶 → 发现损坏 → 拍照留证 → 客户确认 → 扣押金赔偿
```

### 3.3 欠款超期

```
欠款产生 → 7天提醒 → 15天提醒 → 30天提醒 → 限制下单/催款
```

## 5. 导航与入口说明

### 5.1 底部导航（3-Tab）

| Tab | 核心功能 | 入口说明 |
|-----|---------|---------|
| **首页** | 收入展示、快速开单、下一个配送 | 高频操作，每天使用 |
| **客户** | 客户列表、客户档案、客户筛选 | 客户管理入口 |
| **我的** | 库存管理、安检记录、财务对账、个人设置 | 所有低频管理功能入口 |

### 5.2 重要入口位置

| 功能 | 入口位置 | 说明 |
|-----|---------|------|
| 库存管理 | 我的-库存管理 | 采购、充装、盘点 |
| 气瓶充装 | 我的-库存管理-气瓶充装 | 空瓶送充、取回入库 |
| 安检记录 | 我的-安检记录 | 查看历史安检，非主动入口 |
| 日结对账 | 我的-今日对账/财务 | 每日收支核对 |
| 欠瓶管理 | 客户-欠瓶筛选/我的-欠瓶提醒 | 多渠道触达 |

> **设计原则**：除首页、客户两个高频Tab外，所有管理功能统一聚合到"我的"页面，避免导航复杂化。
