## 1. 关键认知洞察

| 传统认知 | 本质认知 | 对产品的影响 |
|---------|---------|-------------|
| 配送员是"配送员" | 配送员是"小老板"：垫资、担风险、控客户 | 关心利润和现金流，而非单纯效率 |
| 订单由用户发起 | 订单由配送员主动：他挑客户、挑时间、挑价格 | 系统必须讨好配送员，而非管理他 |
| 燃企→配送员→用户（线性） | 用户↔配送员↔燃企/其他气源（配送员是枢纽） | 配送员可跳过燃企，平台保持中立 |
| 安检是负担 | 安检是强制要求，必须自动化 | 系统要"代做"而非"要求做" |
| 库存只是数字 | 库存是现金流：空瓶=押金，重瓶=成本 | 必须实时准确 |
| 欠瓶是风险点 | 送重瓶未收回空瓶=资金占用+资产风险 | 必须跟踪和追收 |

## 2. 用户画像

### 2.1 典型用户：老张

- **年龄**：45-55岁
- **经验**：10-20年配送经验
- **规模**：日均配送20-50瓶
- **设备**：千元安卓机，会用但不会打字很快
- **痛点**：算账麻烦、欠瓶记不清、找路费时间

### 2.2 老张的一天

| 时间 | 场景 | 核心需求 | 可视化方案 |
|-----|------|---------|-----------|
| 07:00 | 开工准备 | 看今天要送多少单、预计收入 | 首页待办+收入预测 |
| 09:00 | 装车出发 | 规划最优配送路线 | 地图模式一键规划 |
| 10:30 | 配送中 | 单手操作、快速确认 | 驾驶模式大按钮+语音 |
| 12:00 | 午餐算账 | 看上午赚了多少、谁还欠钱 | 实时收入统计+欠款提醒 |
| 18:00 | 收工 | 对账、算利润 | 日结报表一键生成 |

## 3. 核心设计原则

### 3.1 单手操作优先
- 所有高频操作必须支持单手完成
- 按钮最小尺寸：56×56dp
- **驾驶模式**：全屏超大按钮，只保留核心操作

### 3.2 3秒原则
- 查看核心数据（今日收入）：1秒
- 完成配送确认：1秒
- 查看客户欠款：2秒
- 快速开单：3秒（预选客户+确认）

### 3.3 离线优先
- 车库/地下室无信号也能用
- 有网时自动同步
- 操作永不卡顿

### 3.4 容错设计
- 配送员可能手抖点错
- 关键操作可撤销（5秒内）
- 误删订单可恢复（24小时内）

### 3.5 配置下放
- 押金/租金/残液等业务规则下放至配送员端
- 配送员自主设定，平台只保留法规相关配置

## 4. 可视化设计原则

### 4.1 数据可视化

| 场景 | 传统方式 | App可视化方案 |
|-----|---------|-------------|
| 看收入 | 翻笔记本计算 | 数字卡片+趋势图，红绿标识涨跌 |
| 看欠瓶 | 脑子里记 | 列表+颜色区分（绿7天内/黄8-30天/红超30天） |
| 看路线 | 凭记忆 | 地图标记+路线热力图，显示配送密度 |
| 看库存 | 数瓶子 | 仪表盘形式，低于安全线变红提醒 |

### 4.2 地图模式深化设计

**地图模式不只是导航，是"配送指挥中心"**：

```
┌─────────────────────────────────────────┐
│  🗺️ 地图模式                    [列表]  │
├─────────────────────────────────────────┤
│                                         │
│     🏠                 🏠               │
│          🚚  →  →  →                   │
│     🏠                 🏠               │
│                                         │
│  图例：🔴 待配送  🟢 已完成  🟡 当前位置 │
├─────────────────────────────────────────┤
│  智能路线：已优化 节省15分钟 节省3公里   │
│  [开始配送]  [重新规划]  [添加顺路单]    │
└─────────────────────────────────────────┘
```

**地图模式功能**：
1. **客户分布热力图**：一眼看出客户密集区
2. **智能路线规划**：考虑实时路况、客户预约时间
3. **顺路单推荐**：途中显示附近可顺路配送的订单
4. **配送进度追踪**：已完成/待配送客户地图标记

## 5. 差异化优势

### 5.1 vs 传统纸笔

| 维度 | 纸笔 | App |
|-----|------|-----|
| 速度 | 手写2分钟/单 | 15秒/单 |
| 算账 | 算不清 | 自动计算 |
| 提醒 | 靠记忆 | 自动提醒 |
| 统计 | 无法统计 | 多维度报表 |

### 5.2 vs 现有LPG App

| 维度 | 现有App | 我们的App |
|-----|--------|----------|
| 定位 | 管理系统 | 数字档口 |
| 交互 | 表单式（像办公） | 卡片式（像工具） |
| 语音 | 无 | 支持（付费） |
| 路线 | 无 | 智能规划（付费） |
| 价格 | 年费¥300-500 | 基础免费+按需付费 |

## 6. 风险与应对

| 风险 | 可能性 | 影响 | 应对 |
|-----|-------|------|------|
| 配送员不接受线上化 | 中 | 高 | 保留纸笔习惯，提供"拍照记账"过渡；免费降低门槛 |
| 燃企抵触 | 中 | 中 | 定位"中立平台"，燃企也是货源之一 |
| 数据安全/隐私 | 中 | 高 | 本地存储优先，加密传输；电子签名作为配置 |
| 安检合规风险 | 中 | 高 | 默认最严格标准，拍照留痕；免责声明 |
