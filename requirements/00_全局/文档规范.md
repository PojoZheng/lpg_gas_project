# 文档规范说明

## 为什么需要三个文档？

### 文档定位对比

| 文档 | 定位 | 读者 | 内容特点 | 更新频率 |
|-----|------|------|---------|---------|
| **README.md** | 模块总览 | 所有人（产品、开发、测试、老板） | 一页纸，快速了解模块 | 少（架构变更时） |
| **需求.md** | 功能需求 | 产品经理、开发、测试 | 有什么功能、业务规则 | 中（功能迭代时） |
| **规格.md** | 实现规格 | 开发、设计师 | 怎么做、交互细节 | 多（开发过程中） |
| **数据模型.md** | 数据设计 | 后端开发、DBA | 实体定义、表结构、关系 | 中（需求确定后） |
| **接口.md** | API定义 | 前后端开发 | 请求/响应格式 | 多（开发过程中） |

### 类比理解

就像造房子：
```
README.md  = 楼盘宣传页（一眼知道这个楼盘有什么）
需求.md    = 户型图纸（这个户型有几间房、功能布局）
规格.md    = 施工图纸（墙体厚度、电线走向、施工细节）
```

### 五个文档的关系

```
README.md（入口）
    ├── 模块定位、核心功能一览
    ├── 页面流程图（有哪些页面，怎么跳转）
    ├── 与其他模块的关系
    └── 文档清单（链接到其他文档）
            │
            ├──→ 需求.md（What - 做什么）
            │       ├── 功能列表
            │       ├── 业务规则（欠瓶怎么处理、押金怎么算）
            │       ├── 字段定义
            │       └── 计算逻辑
            │
            ├──→ 规格.md（How - 怎么做）
            │       ├── 交互流程图（点击A后发生什么）
            │       ├── 状态机（订单有哪些状态，怎么流转）
            │       ├── 动画参数（多久、什么缓动函数）
            │       ├── 边界条件（没有网络怎么办、金额太大怎么办）
            │       └── 性能指标
            │
            ├──→ 数据模型.md（Data - 数据如何存储）
            │       ├── 实体定义
            │       ├── 表结构
            │       ├── 字段类型
            │       └── 实体关系
            │
            └──→ 接口.md（API - 前后端如何通信）
                    ├── 接口列表
                    ├── 请求参数
                    └── 响应格式
```

### 会不会冲突？

**原则：单一数据源，分层引用**

| 冲突场景 | 解决方案 |
|---------|---------|
| 需求改了，规格没改 | 规格必须跟随需求更新，以需求为准 |
| README和功能对不上 | README只写概览，具体功能链接到需求 |
| 三个文档都写同一个字段定义 | 字段定义只在需求写，规格引用 |

**避免冲突的方法**：
1. **不重复内容**：字段定义只在需求写，规格不写
2. **链接引用**：规格里提到"详见需求.md#字段定义"
3. **版本同步**：需求变更时，必须检查规格是否需要同步

### 什么时候看哪个文档？

| 角色 | 场景 | 看哪个文档 |
|-----|------|-----------|
| **产品经理** | 写需求 | 先看README结构，写需求.md |
| **开发** | 了解模块 | README（5分钟）→ 需求（30分钟） |
| **开发** | 写代码 | 需求（逻辑）+ 规格（交互细节） |
| **设计师** | 设计界面 | 需求（功能）+ 规格（动画、手势） |
| **测试** | 写测试用例 | 需求（功能点）+ 规格（边界条件） |
| **老板/其他产品经理** | 了解项目 | 只看README |

### 文档维护责任人

| 文档 | 主要维护人 | 审核人 |
|-----|-----------|--------|
| README.md | 产品经理 | 技术负责人 |
| 需求.md | 产品经理 | 开发负责人 |
| 规格.md | 开发负责人 | 产品经理（确认符合需求） |
| 数据模型.md | 后端负责人 | 架构师 |
| 接口.md | 前后端开发 | 技术负责人 |

## 文档模板速查

### README.md 模板

```markdown
# 模块名

## 1. 模块定位
一句话描述这个模块是做什么的。

## 2. 页面流程图
[流程图]

## 3. 核心功能列表
| 功能 | 说明 | 优先级 |

## 4. 与其他模块的关系
### 依赖
- [模块A](../模块A/)

### 被依赖
- [模块B](../模块B/)

## 5. 文档清单
- [需求.md](./需求.md) - 功能需求
- [规格.md](./规格.md) - 详细规格
```
