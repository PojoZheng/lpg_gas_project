# 财务模块规格文档

## 1. 页面清单

| 页面 | 路径 | 说明 |
|-----|------|------|
| 财务首页 | `/finance` | 今日收支、对账入口 |

| 历史对账页 | `/finance/history` | 历史对账记录 |

## 2. 财务首页

### 2.1 页面入口（与首页对齐）

**入口点**（**无底部tab**）：
1. **首页收入卡片** - 点击今日已收/待收卡片 → 进入财务首页（最常用）
2. **我的页面入口** - 我的 → 财务管理（查看历史/统计）

**产品经理视角**：
- 财务是「查看+对账」功能，不是高频操作
- 配送员每天配送结束后看一眼今天赚了多少
- 首页卡片已经满足80%的需求（看数字），深度功能放「我的」

**配送员视角**：
- 最关心的：今天收了多少钱、还有多少没收
- 每周/每月看一眼统计，平时不需要天天看明细

### 5.2 利润计算公式

**单瓶毛利**：
```
单瓶毛利 = 销售单价 - 进货单价

例：
- 15kg售价：¥120
- 15kg进价：¥85
- 单瓶毛利：¥35
```

**日毛利**：
```
日毛利 = 日气款收入 - 日销售瓶数 × 对应进价

例：
- 日气款收入：¥1,050
- 销售：15kg×8瓶(¥85进价) + 10kg×1瓶(¥60进价)
- 销售成本：8×85 + 1×60 = ¥740
- 日毛利：¥1,050 - ¥740 = ¥310
```

**日净利**：
```
日净利 = 日毛利 + 其他收入 - 其他支出

其他收入：押金、租金、残液
其他支出：退押金、损耗
```

## 8. 核心交互

### 8.1 自动记账流程

```
订单完成
    ↓
根据订单类型自动记账：

气款收入 ──► 记录IncomeRecord ──► type=GAS
押金收入 ──► 记录IncomeRecord ──► type=DEPOSIT
租金收入 ──► 记录IncomeRecord ──► type=RENT
残液收入 ──► 记录IncomeRecord ──► type=RESIDUAL

同时：
更新DailySettlement汇总
更新客户财务统计
```

### 8.2 对账确认

```
每日首次进入财务首页
    ↓
检查昨日是否已对账
    ↓
未对账 ──► 显示提醒："昨日财务未对账，请及时核对"
    ↓
点击进入日结对账
    ↓
核对数据
    ↓
确认无误点击[完成对账]
    ↓
锁定昨日数据，生成对账记录
```
