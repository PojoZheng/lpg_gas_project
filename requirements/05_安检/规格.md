## 2. 随瓶安检页面

### 2.1 页面入口（首页无入口）

**触发时机**：
1. **订单完成后自动弹出**（主入口）- 订单完成 → 弹出安检 → 可跳过
2. **我的页面入口** - 我的 → 安检记录（查看历史/补录）

**首页无安检入口** - 安检不是主动功能，是配送流程的一部分

**产品经理视角**：
- 安检是「配送流程的附属动作」，不是独立功能，首页不放入口
- 配送员不想多记一件事，必须跟着订单走
- 政府要数据，我们帮配送员自动上报，不增加负担

**配送员视角**：
- 送完气顺手拍个照，几秒钟的事
- 不需要在首页找安检入口，送完自动弹出

### 2.3 检查项定义

| 检查项 | 正常 | 异常选项 | 是否必填 |
|-------|------|---------|---------|
| 气瓶外观 | ✓ | 变形、锈蚀、泄漏 | 是 |
| 阀门状况 | ✓ | 漏气、损坏、松动 | 是 |

### 2.4 AI识别区域

**拍照后自动识别**：
```
拍照完成
    ↓
上传图片到AI识别服务
    ↓

│  ─────────────────────────────────────  │
│  气瓶编号: LB202601001  ✓ 识别成功      │

│                                         │
│  [确认无误]  [重新识别]  [手动输入]      │
```

**识别失败处理**：
```
识别失败 / 置信度 < 80%
    ↓

    ↓
选项：
- [重新拍照]
- [手动输入编号]
- [跳过识别]
```

### 5.2 任务生成逻辑

```javascript
// 定期安检任务生成
function generateInspectionTasks() {
  const customers = getAllActiveCustomers();
  const tasks = [];
  
  for (const customer of customers) {
    const lastInspection = getLastInspection(customer.id);
    const daysSinceLast = daysBetween(lastInspection?.date, today);
    
    // 90天周期
    if (daysSinceLast >= 75) {
      tasks.push({
        customer_id: customer.id,
        due_date: addDays(lastInspection?.date || customer.created_at, 90),
        days_remaining: 90 - daysSinceLast,
        status: daysSinceLast >= 90 ? 'OVERDUE' : 'PENDING'
      });
    }
  }
  
  return tasks.sort((a, b) => a.days_remaining - b.days_remaining);
}
```

## 6. 数据上报机制

### 6.1 上报流程

```
安检完成提交
    ↓
本地存储记录
    ↓
检查网络状态
    ↓
有网络 ──► 立即上报 ──► 上报成功 ──► 更新状态
    ↓
无网络 ──► 标记待同步 ──► 后台定时重试
```

## 7. 核心交互

### 7.1 安检完成流程

```
订单完成
    ↓
弹出安检页面（可选3秒后自动跳过）
    ↓
┌─────────────────────────────────────┐
│ 选择：                              │
│ ① [立即安检] - 进入安检页面         │
│ ② [稍后补录] - 标记待补录，返回首页 │
│ ③ [无需安检] - 跳过（记录原因）     │
└─────────────────────────────────────┘
    ↓
完成安检
    ↓
拍照（必须）
    ↓
提交
    ↓
本地保存 + 尝试上报
    ↓
Toast "安检记录已保存"
```

### 7.2 跳过安检

```
点击[跳过]
    ↓
弹出原因选择：
- 客户不在家
- 客户拒绝安检
- 下次配送时补做
- 其他（填写原因）
    ↓
记录跳过原因
    ↓
返回首页，该订单标记"未安检"
```

### 7.3 补录安检

```
首页/订单列表看到"未安检"标记
    ↓
点击进入订单
    ↓
[补录安检] 按钮
    ↓
进入安检页面（同上）
    ↓
提交后更新订单状态
```
