# 任务 PRD：认证数据持久化

## 任务目标
将当前认证内存实现升级为可持久化方案，避免服务重启后验证码、会话和令牌丢失。

## 业务背景
当前 `01-auth-session` 任务已完成基础登录链路，但认证数据仅保存在内存，无法满足连续运行与稳定回归需求。

## 范围边界
### 本任务包含
1. 验证码记录持久化
2. 会话记录持久化
3. Access/Refresh 令牌持久化
4. 启停后会话恢复策略

### 本任务不包含
- 新增登录方式（如微信登录）
- UI 页面改版
- 平台端权限体系

## 设计与规范约束
- 接口结构保持与现有客户端兼容
- 错误码与返回格式遵循 shared 契约
- 若涉及界面提示，必须中文且禁用 emoji

## 关联需求
- `requirements/08_系统/登录认证/需求.md`
- `requirements/08_系统/规格.md`
- `requirements/08_系统/数据模型.md`

## 验收标准
- 重启后关键会话可恢复
- 登录和刷新接口向后兼容
- 自动化测试通过
- 完成记录写入 `completion.md`
