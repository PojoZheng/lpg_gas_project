# Task-21 完成说明（退出登录流程与会话收口）

## 本轮目标
- 补齐退出登录入口、会话清理与失效会话拦截提示。
- 保持既有接口契约不变，仅调整前端认证交互与状态处理。

## 实现内容
1. **退出登录入口与反馈**
   - 在 `workbench` 顶部新增“退出登录”按钮。
   - 在 `my` 页面新增顶部与列表的“退出登录”入口。
   - 退出后统一跳转登录页，并通过 `?from=` 展示中文反馈（如“您已安全退出”）。
2. **会话清理完整**
   - `auth-client.js` 新增 `clearCurrentSession()` 与 `logoutCurrentSession()`。
   - 退出时清理本地 `auth_session`，确保 AccessToken/RefreshToken/会话缓存一起移除。
3. **失效会话访问受限页处理一致**
   - `auth-client.js` 新增 `ensureAuthenticatedPage()`：受保护页面在入口校验，未登录统一跳转登录页。
   - 新增 `authFetchJson()`：请求出现 401 / `AUTH_401` 时自动清理会话并统一跳转登录页。
   - 在 `workbench / quick-order / delivery-complete / finance-daily-close / my` 页接入入口拦截。
4. **接口契约保持不变**
   - 未新增、删除或修改后端 API 路径与请求响应字段契约。
   - 仅增强前端本地会话处理与页面跳转逻辑。

## 场景验证
- 场景 1：工作台点击“退出登录”
  - 结果：清理本地会话，跳转登录页并提示“您已退出登录”。
- 场景 2：我的页点击“退出登录”
  - 结果：清理本地会话，跳转登录页并提示“您已安全退出”。
- 场景 3：本地会话清空后直接访问受保护页
  - 结果：统一拦截并跳转登录页，提示“请先登录后再继续操作”。
- 场景 4：服务端返回会话失效（401/AUTH_401）
  - 结果：统一清理本地会话并跳转登录页，不出现“假登录”停留状态。
