## /critique 发现（视觉一致性与信息层级）
1. **视觉 token 重复定义**：三页均内嵌相似变量与按钮样式，存在字号、间距、按钮高度微差，导致跨页一致性不足。
2. **信息层级不稳定**：页面标题与卡片标题层级不完全一致，关键状态（同步状态/安检状态）在不同页面视觉权重不统一。
3. **主次按钮体系偏散**：同类操作在不同页面颜色、边框、文案长度不一，认知成本上升。
4. **反馈区域位置不统一**：提示文案（tip）在不同页面位置与样式差异大，失败反馈难以快速扫描。

## /audit 发现（可用性、响应式、交互安全）
1. **移动端可触达性可继续收敛**：少量按钮仍使用 38~44 高度，建议统一不低于 48 触达目标。
2. **390x844 视口下局部拥挤风险**：表单密集区和双列按钮区在长文案或错误态时可能挤压，需统一断点策略。
3. **返回路径策略需标准化**：不同页面“返回工作台”入口位置与交互策略不完全一致。
4. **状态回退链路需统一文案**：同步失败、安检失败、登录失败虽有提示，但“下一步动作”文案模板尚未统一。

## 最小安全改动建议（本轮仅清单，不大改）
- 抽离一份共享 UI token（颜色、字号、间距、圆角、按钮高度）并逐页替换硬编码。
- 建立统一按钮规范（primary/secondary/warn）与统一提示条样式（info/success/error）。
- 统一移动端断点：`max-width: 420px` 下输入区/按钮区单列化策略。
- 统一返回入口位置与文案（顶部返回 + 底部关键回退入口按任务场景择一固定）。
- 为失败态补充“问题 + 下一步动作”统一模板，减少歧义。
