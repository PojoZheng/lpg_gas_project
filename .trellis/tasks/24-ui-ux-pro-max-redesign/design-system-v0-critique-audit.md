# Task-24 Design-System v0 /critique + /audit

## 范围
- delivery-app 与 platform 现有页面样式基线（跨端一致性）
- 基础组件复用可能性（按钮、卡片、导航、表单、图标）

## /critique 关键结论
1. 两端存在大量重复 token 与组件样式定义，维护成本高且容易漂移。
2. 同语义组件在 app/web 命名不统一，A/B 并行改造难以复用。
3. 图标体系缺少共享载体，页面通常以文案替代，视觉识别弱。

## /audit 关键结论
1. 组件状态覆盖不完整（禁用/错误/加载），易造成交互反馈不一致。
2. 低层样式未形成可复用入口，后续改造会频繁复制粘贴。
3. 缺少最小接入示例，A/B 无法快速落地验证。

## 落地动作（v0）
- 建立 `/.trellis/ui-kit` 共享目录并交付 tokens/components/icons/example。
- 规范首版只包含基础组件层，不触碰业务接口与流程逻辑。
- 更新 spec 使后续页面改造必须优先复用 UI Kit。
