# Task 44：订单域退货换货逆向流程

## 背景
配送过程中可能出现气瓶质量问题或客户原因需要退货换货。本任务实现两种逆向流程：
1. **当场换货** - 交易完成后5分钟内快捷换货
2. **24小时内退货** - 已完成订单的退货申请

## 需求来源
- `requirements/01_工作台/需求.md` §12 退货/换货逆向流程

## 功能规格

### 1. 当场换货（5分钟内）

#### 触发条件
- 订单状态为"已完成"
- 完成时间在5分钟内

#### 流程
```
完单成功页面显示[快捷换货]按钮（5分钟倒计时）
    ↓
点击[快捷换货]
    ↓
1. 原订单状态变更为"换货中"
2. 原订单库存回滚（重瓶+1，标记待检）
3. 弹出换货面板
    ↓
选择规格/数量（默认同原订单）
    ↓
选择差价处理方式
    ↓
完成换货 → 创建新订单（已完成状态）
```

#### 换货面板
| 字段 | 说明 |
|-----|------|
| 原订单信息 | 客户、规格、数量、金额 |
| 新气瓶规格 | 15kg/10kg/50kg 单选 |
| 新气瓶数量 | 数字输入，默认1 |
| 差价处理 | 无需补差/客户补差/退客户差 |
| 完成换货按钮 | 校验通过后启用 |

#### API
```
POST /orders/:id/exchange
Request:
{
  newSpec: "15kg",
  newQuantity: 1,
  priceDiff: 0,  // 正数=客户补差，负数=退客户差
  priceDiffReason: "规格不同"
}

Response:
{
  success: true,
  data: {
    originalOrder: { id, status: "exchanging" },
    newOrder: { id, status: "completed" },
    inventoryChange: { spec, delta: -1 }
  }
}
```

### 2. 24小时内退货

#### 触发条件
- 订单状态为"已完成"
- 完成时间在24小时内

#### 流程
```
订单详情页
    ↓
显示[申请退货]按钮（24小时内）
    ↓
点击[申请退货]
    ↓
退货面板
    ↓
选择退货原因
确认收回气瓶
确认退款金额
选择退款方式
    ↓
确认退货
    ↓
原订单标记为"已退货"
库存回滚（重瓶+1，待检）
收入扣除
```

#### 退货面板
| 字段 | 说明 |
|-----|------|
| 原订单信息 | 客户、规格、数量、金额、支付方式 |
| 退货原因 | 质量问题-阀门漏气/质量问题-瓶身损坏/客户原因-不需要了/其他 |
| 收回气瓶确认 | 扫码确认/手动确认 |
| 退款金额 | 默认实收金额，可修改 |
| 退款方式 | 现金/原路退回 |
| 确认退货按钮 | 校验通过后启用 |

#### API
```
POST /orders/:id/return
Request:
{
  reason: "质量问题-阀门漏气",
  reasonDetail: "",
  bottleReturned: true,
  bottleBarcode: "",
  refundAmount: 130,
  refundMethod: "cash",  // cash, original
  note: ""
}

Response:
{
  success: true,
  data: {
    order: { id, status: "returned" },
    refundRecord: { id, amount: 130 },
    inventoryChange: { spec: "15kg", delta: 1 }
  }
}
```

### 3. 订单状态扩展

新增状态：
| 状态 | 编码 | 说明 |
|-----|------|------|
| 换货中 | exchanging | 正在执行换货流程 |
| 已退货 | returned | 退货已完成 |

状态流转：
```
completed → exchanging → completed (新订单)
completed → returned
```

---

### 4. 数据影响

| 操作 | 订单状态 | 库存影响 | 收入影响 | 客户账户 |
|-----|---------|---------|---------|---------|
| 当场换货 | 原：换货中 → 新：已完成 | 原瓶回库（待检），新瓶出库 | 按差价调整 | 不变 |
| 24小时内退货 | 原：已退货 | 重瓶回库（待检） | 今日已收扣除 | 如有存款，优先退存款 |

---

### 5. 库存处理

换货/退货时库存变动：
```
原订单消耗的重瓶：+1（回到库存，状态待检）
换货时新订单消耗：-1（正常出库）
```

库存日志类型：
- `exchange_return` - 换货回库
- `order_return` - 退货回库
