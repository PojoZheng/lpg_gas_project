# Task 46：财务域 P1 - 收入明细与日结对账

## 1. 背景与目标

### 1.1 业务背景
工作台首页已展示「今日已收/待收」，但：
- 毛利数据为占位（无成本接口）
- 无法查看历史收入明细
- 收工后无法核对现金/微信/支付宝是否一致

### 1.2 目标
- 提供收入明细查询（时间维度 + 分类维度）
- 提供日结对账功能（收工后核对）
- 为后续「毛利计算」奠定数据基础

### 2.3 日结对账页（finance-daily-close.html）

#### 2.3.1 页面结构

```
┌─────────────────────────────────────────┐
│  [← 返回]  日结对账    2026年4月12日     │
├─────────────────────────────────────────┤
│                                         │
│  【今日收支】                            │
│  收入          +¥1,680                  │
│  支出          -¥0（本期不实现支出录入） │
│  ─────────────────────                  │
│  净利润        +¥1,680                  │
│                                         │
├─────────────────────────────────────────┤
│  【订单汇总】                            │
│  完成订单      8单                      │
│  配送气瓶      15kg×6, 10kg×2           │
│  收回空瓶      15kg×5, 10kg×1           │
│  产生欠瓶      2个                      │
│                                         │
├─────────────────────────────────────────┤
│  【收款方式核对】                        │
│  现金          ¥980    [实收: ____] [✓] │
│  微信          ¥450    [实收: ____] [✓] │
│  支付宝        ¥150    [实收: ____] [✓] │
│  记账（欠款）  ¥100    （无需核对）      │
│                                         │
├─────────────────────────────────────────┤
│  【对账结果】                            │
│  差异：¥0    状态：平账 ✓               │
│                                         │
│     [  确 认 日 结  ]                   │
│                                         │
└─────────────────────────────────────────┘
```

#### 2.3.2 对账流程

```
进入页面
    ↓
加载今日数据（收入、订单、收款方式汇总）
    ↓
用户输入各渠道「实收金额」
    ↓
系统自动计算差异
    ↓
差异为0 → [确认日结] 按钮可用
差异不为0 → 提示差异原因（可强制确认）
    ↓
点击确认 → 生成日结记录 → 提示「日结完成」
```

#### 2.3.3 边界情况

| 场景 | 处理 |
|------|------|
| 差异不为0 | 提示「差异¥XX，请检查」，提供「强制确认」按钮（记录差异原因） |
| 已日结过 | 显示「今日已日结」，提供「重新日结」（覆盖）或「查看记录」 |
| 无订单 | 提示「今日无订单，无需日结」 |

## 3. 接口设计

### 3.1 GET /finance/income

查询收入明细

**请求参数：**
```json
{
  "startDate": "2026-04-12",
  "endDate": "2026-04-12",
  "type": "all|gas|deposit|rent|residual|recharge",
  "page": 1,
  "size": 20
}
```

**响应：**
```json
{
  "success": true,
  "data": {
    "summary": {
      "totalIncome": 1680,
      "byType": {
        "gas": 1320,
        "deposit": 150,
        "residual": 25,
        "other": 185
      }
    },
    "items": [
      {
        "id": "FIN-001",
        "time": "10:30",
        "customerName": "李大妈",
        "type": "gas",
        "typeLabel": "气款",
        "amount": 120,
        "method": "cash",
        "methodLabel": "现金",
        "orderId": "ORD-001"
      }
    ],
    "pagination": {
      "page": 1,
      "size": 20,
      "total": 15,
      "hasMore": false
    }
  }
}
```

### 3.2 GET /finance/daily-close

获取日结对账数据

**请求参数：**
```json
{
  "date": "2026-04-12"
}
```

**响应：**
```json
{
  "success": true,
  "data": {
    "date": "2026-04-12",
    "income": {
      "total": 1680,
      "byMethod": {
        "cash": 980,
        "wechat": 450,
        "alipay": 150,
        "credit": 100
      }
    },
    "orders": {
      "completed": 8,
      "bottlesDelivered": {"15kg": 6, "10kg": 2},
      "bottlesReturned": {"15kg": 5, "10kg": 1},
      "bottlesOwed": 2
    },
    "status": "pending|confirmed",
    "confirmedAt": null,
    "lastConfirmedBy": null
  }
}
```

### 3.3 POST /finance/daily-close/confirm

确认日结

**请求体：**
```json
{
  "date": "2026-04-12",
  "actualAmounts": {
    "cash": 980,
    "wechat": 450,
    "alipay": 150
  },
  "difference": 0,
  "differenceReason": "",
  "force": false
}
```

**响应：**
```json
{
  "success": true,
  "data": {
    "recordId": "DC-20260412-001",
    "confirmedAt": "2026-04-12T22:30:00Z"
  }
}
```

## 4. 数据模型（后端）

### 4.1 FinanceEntry（收入记录）

基于现有 `server.js` 中的 `financeEntries` 扩展：

```javascript
{
  id: "FIN-xxx",
  type: "gas|deposit|rent|residual|recharge|return_reversal",
  orderId: "ORD-xxx",
  customerId: "CUST-xxx",
  customerName: "李大妈",
  amount: 120,
  method: "cash|wechat|alipay|transfer|credit",
  createdAt: timestamp,
  date: "2026-04-12"  // 用于按天查询
}
```

### 4.2 DailyCloseRecord（日结记录）

```javascript
{
  id: "DC-20260412-001",
  date: "2026-04-12",
  incomeSummary: {
    total: 1680,
    byMethod: {...}
  },
  actualAmounts: {
    cash: 980,
    wechat: 450,
    alipay: 150
  },
  difference: 0,
  differenceReason: "",
  confirmedAt: timestamp,
  confirmedBy: "userId"
}
```
