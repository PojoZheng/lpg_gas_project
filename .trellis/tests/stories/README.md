# Story Tests

本目录用于编写以用户故事为中心的自动化测试，而不是只验证单个接口或页面是否存在。

## 编写原则

- 一个测试文件尽量覆盖一个主故事
- 测试步骤按用户操作顺序编排
- 断言同时覆盖：
  - 页面是否可达
  - 关键数据是否更新
  - 页面之间是否保持一致
  - 异常反馈是否符合用户预期

## 推荐命名

- `story_<domain>_<goal>.py`
- 例如：`story_customer_debt_collection.py`

## 推荐结构

1. 角色与前置条件
2. 主路径步骤
3. 异常路径步骤
4. 关键断言
5. 失败归因说明
