# 工作台域概述

## 目标
让配送员在首页快速确认“今日收入、待送任务、下一步动作”。

## 核心流程
1. 加载今日已收/待收
2. 展示下一个配送客户卡片
3. 点击快速开单进入订单流程

## 关键规则
- 待配送排序：预约时间 > 创建时间 > 距离
- 首页只做聚合展示，不承载重配置
- 无网时展示本地缓存；检测到“离线待同步且当前联网”时，进入首页触发一次同步模态框

## 依赖
- 订单域、财务域、系统域同步状态

## MVP 范围
- 收入卡、待送卡、快速开单入口、下拉刷新
- 工作台仅承载高频入口；低频入口统一通过“我的”聚合页进入

## Task-16 规范收口（A/B 改造对齐）
- 首页聚焦两块高频主操作：`快速开单`、`待配送`，不再堆叠低频管理入口。
- 工作台信息块收敛为：
  - 今日收入主摘要（总览 + 已收/待收）
  - 下一待办（客户/地址/时间/订单号）
- 底部导航固定三项：`首页 / 客户 / 我的`，其中“我的”为低频能力统一入口。
- 登录与切换账号保留在工作台顶部右侧，作为系统级入口而非业务主操作。
- 工作台保留下拉刷新与自动刷新机制，提示文案使用中文且无 emoji。

## 产品评审优化清单（V1 -> V1.1）
以下优化项用于提升“3 秒可见核心数据、3 步完成核心操作”的落地质量。

### P0（下一迭代必须进入）
1. **关键动作闭环**
   - 工作台“快速开单”按钮点击后必须进入真实开单流程，不允许仅提示“待实现”。
   - 若能力未开启，需展示“原因 + 下一步动作”（例如：去客户列表先建档）。
2. **同步触发策略优化**
   - 首页取消常驻同步状态卡，避免占用首页主流程注意力。
   - 当“存在离线待同步数据且网络可用”时，进入首页自动弹出同步模态框。
   - 模态框至少包含“立即同步/稍后处理”两个动作与待同步条数信息。
   - 同步失败后提供“查看同步明细/重试”入口，不在常态下常驻展示。
3. **待配送卡片可行动**
   - “下一待配送”卡片应至少支持“查看详情”和“开始配送”两个动作入口，避免只读信息。

### P1（建议进入后续迭代）
1. **空状态与异常状态规范化**
   - 无待配送时给出明确引导（例如“去快速开单”）。
   - 收入数据为空时说明口径（例如“今日暂无完单收款”）。
2. **信息可信度增强**
   - 收款卡增加“统计时间点”或“最近更新时间”，避免用户误判数据实时性。
3. **刷新反馈一致性**
   - 下拉刷新与按钮刷新应统一反馈文案和加载态，避免重复/冲突提示。

### 设计约束补充
- 所有新增提示文案必须中文，且禁用 emoji。
- 首页只做聚合与分发，不新增重表单输入。
- 主色继续使用 `#4799a0`，状态色仅用于辅助表达，不替代主操作色。
