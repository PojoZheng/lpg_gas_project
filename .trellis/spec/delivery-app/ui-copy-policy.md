# 界面中文文案规范

## 1. 总体原则
- 界面文案统一中文。
- 术语统一来源：`requirements/00_全局/术语表.md`。
- 文案应短句、可执行、避免含糊表达。

## 2. 命名规则
- 页面标题：动词或名词短语，例如“快速开单”“今日对账”。
- 按钮文案：动词开头，例如“提交订单”“确认完单”“稍后处理”。
- 状态文案：简洁明确，例如“待同步”“同步失败”“已完成”。

## 3. 错误与提示
- 错误文案采用“问题 + 建议动作”结构。
- 空状态文案提供明确下一步引导。
- 通知文案先结论后详情，首句不超过 16 字。

## 4. 禁止事项
- 严禁使用 emoji。
- 严禁中英混杂文案（必要技术词除外）。
- 严禁使用带歧义口语化表达。
