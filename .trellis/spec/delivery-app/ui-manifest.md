# 配送员端 UI Manifest

本文件定义配送员端 UI 的产品风格、交互约束和禁令，作为所有界面实现的强制准则。

## 1. 风格目标
- 风格参照参考图：浅色背景、卡片化布局、大圆角、弱边框、少装饰。
- 优先突出“数字信息 + 主操作”，减少视觉干扰。
- 页面文案统一中文，术语统一使用 `requirements/00_全局/术语表.md`。

## 2. 强制禁令
- 严禁在任何界面中使用 emoji。
- 严禁使用重阴影、渐变背景和复杂纹理。
- 严禁中英混杂文案（必要技术词除外）。

## 3. 图标规范
- 图标统一使用矢量图标库（例如 `MaterialCommunityIcons`）。
- 图标应置于统一圆角容器中，使用 `Brand.PrimarySoft` 或中性色浅底。
- 禁止将 emoji 当作业务图标或状态图标。

## 4. 交互规范
- 核心流程（开单、完单、安检、日结）必须 1-2 次触达关键输入区域。
- 所有关键提交动作需有明确反馈：加载中、成功、失败、可重试。
- 离线模式下采用乐观更新，并展示同步状态标签。
- 关键交易动作需支持短时撤销入口（例如 5 秒撤销）。

## 5. 布局规范
- 首页结构：顶部信息摘要、中部任务/功能卡片、底部导航与主操作按钮。
- 单手操作优先：高频按钮置于下半屏。
- 所有点击区域最小为 `48x48dp`。

## 6. 导航收敛规范（task-24）
- 底部导航常驻且仅允许 3 个 Tab：`首页 / 客户 / 我的`。
- 低频能力（财务、同步、配置、平台相关入口）默认收敛到“我的”页面。
- 首页保留高频流程入口（如快速开单、待配送完单），避免堆叠低频管理入口。
