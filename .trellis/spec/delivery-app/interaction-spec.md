# 交互细则规范

## 1. 状态反馈
- 点击提交后 120ms 内给出反馈（按钮 loading 或状态条）。
- 成功反馈采用简短中文提示，不自动跳转超过 1 层。
- 失败反馈必须给出可执行动作（重试、返回修改、联系客服）。

## 2. 离线交互
- 离线状态下允许继续操作，显示“待同步”状态标签。
- 网络恢复后自动重试同步，并更新状态。
- 冲突时展示中文说明与处理选项，不可静默失败。

## 3. 输入交互
- 表单默认提供常用值与最近值。
- 金额输入必须显示单位与格式校验。
- 错误提示用中文、短句、可直接执行修复动作。

## 4. 操作防误触
- 关键动作（完单、删除、回滚）须有二次确认或撤销入口。
- 支持 5 秒撤销关键交易动作。

## 5. 禁令
- 严禁使用 emoji 作为状态提示或按钮内容。
- 严禁仅靠颜色表达错误状态，必须有文字说明。
