# 离线同步协议

## 原则
- 移动端本地先写，后端异步同步，不阻塞用户操作。
- 每条本地写入必须携带 `offline_id` 与 `sync_status`。
- 同步冲突必须可追踪并可人工处理。

## 同步流程
1. 业务写入本地数据并写入 `OfflineQueue`
2. 后台任务批量提交变更
3. 服务端按实体类型执行幂等写入
4. 返回冲突/成功结果
5. 客户端更新 `sync_status` 与 `DataSyncLog`

## 冲突策略
- 订单状态冲突：优先保护已完成事实，产生差异记录。
- 库存冲突：以服务端库存基线校正，并生成异常任务。
- 客户资料冲突：字段级合并，时间戳较新者优先。

## 重试策略
- 网络失败指数退避重试，最大 3 次。
- 达到上限后标记 `failed` 并提示人工处理。
