# 核心架构总纲

## 1. 三端职责
| 端 | 角色 | 核心职责 |
|---|---|---|
| delivery-app | 业务执行端 | 开单、配送、完单、安检、日结 |
| platform | 运营管理端 | 规则配置、监控分析、合规治理 |
| backend | 协调中台 | 同步、审计、上报、策略分发 |

## 2. 全局约束
- 离线优先：本地先写，后台异步同步。
- 统一同步字段：`offline_id`、`sync_status`、`updated_at`。
- 统一安全边界：敏感字段加密，接口全链路 HTTPS。
- 统一可追溯：关键操作必须记录审计日志。

## 3. 数据一致性策略
- 订单为交易主事实，驱动库存、财务、安检联动。
- 同步冲突按实体策略处理：
  - 订单：保护已完成事实
  - 库存：服务端基线校正
  - 客户资料：字段级合并

## 4. 性能底线
- 本地查询 `<100ms`
- 后台同步 `<5s`
- 后端 API `<200ms`
