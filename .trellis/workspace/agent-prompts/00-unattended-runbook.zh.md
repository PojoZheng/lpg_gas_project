# 三窗口「少打扰」运行手册（协调器单点沟通）

目标：**你只跟协调器（`03-dispatcher`）说需求与节奏**；开发 A/B、集成三个窗口在**不卡住**时尽量自己往下做；**只有卡住时**你来处理。

## 1. 做不到、也不应承诺的事

- Cursor 里的 Agent **不是**操作系统后台服务，不能 7×24 无人物理值守。
- **终端审批 / 网络 / 沙箱**若设为每次询问，会打断「连续执行」，需要你在 Cursor 设置里放宽（见下）。

## 2. 三窗口尽量「自己往下跑」的必备条件

1. **模式**：该窗口固定为 **Agent（可执行）**，不要用纯问答。
2. **提示词**：窗口分别贴 `01-dev-coding.zh.md` / `02-integrator.zh.md`，已含 **0.1 自动化续跑**（bootstrap 后禁止只交摘要）。
3. **批准策略**（减少停住等你点）：在 Cursor 设置里对**本仓库**使用「自动跑终端 / 少确认」一类选项（以你本机 Cursor 版本为准），避免每一步都弹确认。
4. **一条总指令**（每个窗口在贴完角色提示词后可加一次，不必每轮重复）：
   - 「除非报错或职责不符，否则**不要停下来问我**；按 task acceptance 连续实现 / 合并 / 预览。」

## 3. 什么叫「卡住」（只有这时需要你）

| 现象 | 谁处理 |
|------|--------|
| 同一条命令失败 ≥2 次 | 把**完整报错**贴到**协调器**会话，由协调器改任务/拆范围/换顺序，再广播给对应窗口 |
| 合并冲突、语义不确定 | **集成窗口**内处理；解决不了 → 协调器决策 |
| 上下文将满 / 已提示新开会话 | 在该 worktree **新开会话**，首条只跑 `session_bootstrap.py` + 贴同一段角色提示词 |
| 在等「是否继续」类问题 | 窗口应已禁止；若仍出现 → 回「继续，按 acceptance 执行」 |
| 预览起不来、端口占用 | 集成窗口看 `start_local_preview.sh` 输出；仍不行 → 协调器 |

## 4. 你不盯屏时的「轻量检查」

- **GitHub Actions**：有 push 就看 CI 红绿（门禁，不是替代开发）。
- **节奏**：每完成一轮（合并进 `main`）协调器收一次 **URL + 完成情况**；中间不必每个文件都看。

## 5. 协调器侧（你只沟通这一处时）

- 所有**范围/优先级/要不要新任务**在协调器会话定；开发/集成窗口**只收任务路径与一句广播**，避免三边传话。
- 下一轮任务用 `suggest_next_task.py` + `coordinator_broadcast.py` 生成指令，减少口头误差。
- 写指针后加 **`--notify`**：本机弹通知，提醒你到三窗口各发一条 `session_bootstrap`（不能替代三窗执行，只是少盯屏）。
